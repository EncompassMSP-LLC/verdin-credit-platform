## 1. Outcome

Lender/LO team can submit referrals, read pipeline status, request readiness packages, and receive digests — with clear advisory boundaries.

## 2. Pre-kickoff

- [ ] MSA / pilot order form signed (counsel)
- [ ] Branch / LO list and roles
- [ ] Success metrics agreed (Vol 06)
- [ ] Claim language reviewed (Vol 18)
- [ ] Data-sharing expectations written

## 3. Kickoff agenda (60 min)

| Time  | Topic                           |
| ----- | ------------------------------- |
| 0–10  | Goals + non-claims              |
| 10–25 | Demo: refer → status → export   |
| 25–40 | Roles, SLAs, digest preferences |
| 40–50 | First 10 referrals plan         |
| 50–60 | Support contacts + Q&A          |

## 4. Access setup

1. Create partnership record (mortgage partner module / CRM)
2. Invite LOs (viewer vs submitter vs admin)
3. Confirm login path (`/lender` or SSO later)
4. Configure digest emails
5. Share Partner Kit PDF

## 5. First-14-days playbook

| Day | Action                               |
| --- | ------------------------------------ |
| 0   | Kickoff                              |
| 1–3 | Each LO submits 1 test referral      |
| 3–7 | Ops processes; LO sees stage updates |
| 7   | Mid-pilot check-in                   |
| 14  | Review metrics; expand or adjust     |

## 6. LO cheat sheet (one page)

- How to refer
- What stages mean
- What “Lending Ready” does / does not mean
- How to ask for package
- Who to contact

## 7. Exit criteria (onboarded)

- [ ] ≥1 LO active
- [ ] ≥3 live referrals
- [ ] Digest or status check confirmed
- [ ] No claim-language violations observed
