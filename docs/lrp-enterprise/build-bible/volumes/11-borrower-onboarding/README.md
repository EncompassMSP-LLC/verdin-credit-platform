## 1. Outcome

Borrower can log in, understand advisory readiness, complete first documents/tasks, and know when staff will next update them — without believing a loan is guaranteed.

## 2. Entry paths

| Path                    | Trigger                                    |
| ----------------------- | ------------------------------------------ |
| Operator invite         | Staff creates client / sends portal invite |
| Partner referral accept | Referral accepted → borrower linked        |
| Self-serve (later)      | Marketing CTA — defer until ops capacity   |

## 3. Day-0 checklist (borrower)

1. Open invite / create password
2. Confirm profile basics
3. Read “What Lending Ready means” (advisory)
4. Upload ID / required docs (as requested)
5. View readiness score / band (or “pending analysis”)
6. Open first task on action plan
7. Optional: start first learning module

## 4. Day-0 checklist (staff)

1. Verify invite delivered
2. Confirm consent / disclosures on file
3. Queue credit analysis if reports available
4. Set expected first check-in date
5. If partner-referred: first status note for LO

## 5. First-week milestones

| Day | Borrower                      | Staff                        |
| --- | ----------------------------- | ---------------------------- |
| 1   | Login + docs started          | Welcome note                 |
| 2–3 | Tasks in progress             | Analysis / findings if ready |
| 5–7 | Learning + checklist progress | Partner update if applicable |

## 6. Scripts (themes)

- Welcome: partnership + readiness journey
- Docs needed: why each item matters
- Score explanation: advisory, multi-factor, not underwriting
- Stuck: how to message staff

## 7. Exit criteria (onboarded)

- [ ] Portal access confirmed
- [ ] Baseline profile complete
- [ ] At least one readiness view or “pending” state understood
- [ ] Next action clear
