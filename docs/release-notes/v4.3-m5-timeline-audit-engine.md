# Release Notes — v4.3 M5: Timeline & Audit Engine

**Date:** 2026-06-29  
**Epic:** Document Intelligence Platform — Milestone 5 / Operational Core

## Summary

Introduces a platform-wide immutable timeline fed by a reusable in-process event bus. Domain services publish typed events; the Timeline module subscribes and persists append-only audit records.

## New capabilities

### Shared packages

- **`packages/event-bus`** — `PlatformEvent`, publish/subscribe registry
- **`packages/event-types`** — typed event constants for case, account, document, auth, and task domains
- **`packages/document-classification`** — the rule-based classification engine, extracted from `apps/api` so the worker can run it directly (previously the worker reached into the API source by filesystem path, which failed in the container image)

### Backend

- Migration `008_timeline_audit_engine` extends `timeline_events` with organization scope, entity links, JSON metadata, performer, and source module
- `GET /timeline` and `GET /timeline/{id}` with filters (case, account, document, type, user, date range)
- Automatic publishing from case, account, document, and auth services
- Worker pipeline emits OCR, classification, metadata, and entity resolution timeline events

### Frontend

- Timeline page with filters and event detail metadata display
- Navigation entry under **Timeline**

## Architecture

```
Service action → publish(PlatformEvent) → EventBus → Timeline subscriber → timeline_events (append-only)
```

Future consumers (workflow automation, analytics, AI agents) can subscribe to the same bus without changing domain services.

## Fixes shipped alongside

- Corrected the M4 `DocumentMetadata` ↔ `Document` relationship back-reference (`extracted_metadata`), which previously broke SQLAlchemy mapper configuration.
- `DocumentResponse.from_model` no longer triggers an async lazy-load for the metadata relationship on freshly uploaded documents.
- Editable shared-package installs in the API and worker Docker images now resolve correctly.
- Worker compose service now receives `DATABASE_URL_SYNC` and MinIO settings (required now that the worker writes timeline events).

## Next

- Operational Dashboard metrics sourced from timeline + processing health
