# Release Notes — v4.3 Operational Dashboard

**Version:** 4.3.0 (Operational Core completion)  
**Date:** 2026-06-29

## Summary

The Operational Dashboard completes Version 4.3 by delivering an **Operations Command Center** — one API call, one response, and a staff UI that answers three questions at a glance: what is happening right now, what needs attention, and how the business is performing.

## Backend

- New `dashboard` module with aggregation repository, service, and router
- **`GET /api/v1/dashboard`** returns:
  - `kpis` — open cases, active accounts, pending tasks, processing counts, OCR queue, average resolution time
  - `processing` — live document pipeline stages (uploads, OCR, classification, metadata, entity resolution)
  - `tasks` — operational work queue (overdue tasks, high-priority cases, review items, OCR failures, unresolved entities)
  - `timeline` — recent activity feed
  - `ai` — classification, metadata, entity resolution, confidence, AI-ready document counts
  - `performance` — business metrics (throughput, resolution rate, cases opened/completed)
- Response includes `generated_at` and `refresh_seconds` for polling; payload contract is WebSocket-ready for future push updates
- RBAC: `read_only+`

## Frontend

- Replaced placeholder dashboard with **Operations Command Center** layout
- Single TanStack Query fetch with `refetchInterval` driven by `refresh_seconds`
- Sections: Executive KPIs, work queue panels, processing health, AI readiness, timeline feed, business metrics

## API Client

- `getDashboard()` in `@verdin/api-client` with full TypeScript types

## Tests

- Dashboard endpoint auth, RBAC, and snapshot shape integration tests

## Version 4.3 freeze

With this epic, the Operational Core is complete. Tag **`v4.3.0`** after merge to mark the cohesive operational platform release. Version 4.5 will focus on automation (import wizard, advanced OCR, workflow, dispute generation) without revisiting foundational architecture.
