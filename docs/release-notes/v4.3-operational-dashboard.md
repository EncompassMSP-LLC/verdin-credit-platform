# Release Notes — v4.3 Mission Control Dashboard

**Version:** 4.3.0  
**Date:** 2026-06-29

## Summary

The Operational Dashboard ships as **Mission Control** — a product-grade operations command center backed by a single aggregation API. The frontend makes one request; the backend owns all metric sourcing.

## API contract

`GET /api/v1/dashboard` returns:

```json
{
  "generated_at": "...",
  "refresh_seconds": 30,
  "overview": {},
  "cases": {},
  "accounts": {},
  "documents": {},
  "timeline": [],
  "tasks": {},
  "processing": {},
  "performance": {},
  "alerts": {}
}
```

## Dashboard cards

| Group        | Metrics                                                                                   |
| ------------ | ----------------------------------------------------------------------------------------- |
| Operations   | Open cases, active accounts, documents, tasks due today, overdue tasks                    |
| Processing   | OCR queue, OCR failed, classification/metadata/entity resolution pending                  |
| Performance  | Cases created/closed today, avg resolution time, documents/accounts per case              |
| Intelligence | Classification confidence, entity resolution confidence, AI-ready, unresolved             |
| Timeline     | Recent activity with case number and document file name                                   |
| Alerts       | OCR failures, unmatched entities, documents requiring review, high-priority overdue tasks |

## Frontend

- Mission Control layout with executive overview strip
- Grouped metric cards (Operations, Processing, Performance, Intelligence)
- Timeline feed + alerts panel
- TanStack Query with `refetchInterval` driven by `refresh_seconds`

## Tests

- Dashboard auth, RBAC, and snapshot shape integration tests
