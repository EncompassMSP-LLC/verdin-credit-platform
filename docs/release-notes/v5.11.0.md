# Release Notes — Version 5.11.0 (Compliance-Gated Production Execution)

**Version:** 5.11.0  
**Date:** 2026-07-06  
**Tag:** `v5.11.0`

## Summary

Version 5.11.0 completes the **Compliance-Gated Production Execution** milestone after v5.10.0. It ships unsupervised bureau re-filing audit, live charge retry execution scaffold, HRIS passwordless UI scaffold, and multi-IdP bulk provisioning scaffold — all behind feature flags and documented **Partial** scope limits.

All four 5.11 epics ship as **Partial ✅**. See [`docs/governance/version-5.11-scope.md`](../governance/version-5.11-scope.md).

## Shipped in v5.11.0

| Area                              | Highlights                                                                         |
| --------------------------------- | ---------------------------------------------------------------------------------- |
| **Unsupervised bureau re-filing** | Operator-gated unsupervised re-filing run audit from refiled bureau re-filing runs |
| **Live charge retry execution**   | Admin-gated live charge retry execution audit from retried charge retry runs       |
| **HRIS passwordless UI**          | Admin-gated HRIS-linked passwordless UI audit from enrolled SAML enrollment runs   |
| **Multi-IdP bulk provisioning**   | Admin-gated bulk IdP provisioning audit from approved HRIS passwordless UI runs    |
| **Governance**                    | Capability matrix 5.11 sign-off, scope doc, completion checklist, release notes    |

## Unsupervised bureau re-filing

Operator-gated unsupervised re-filing run audit from `refiled` bureau re-filing runs:

| Capability | Details                                                                                                                                  |
| ---------- | ---------------------------------------------------------------------------------------------------------------------------------------- |
| Endpoints  | `GET /compliance/bureau-unsupervised-refiling/status`, `GET .../runs`, `POST .../refiling-runs/{id}/start`, `POST .../runs/{id}/approve` |
| Audit      | `bureau_unsupervised_refiling_runs` table (migration `059`)                                                                              |
| Gate       | `ENABLE_BUREAU_UNSUPERVISED_REFILING=true` (requires bureau re-filing readiness)                                                         |
| Deferred   | Live bureau API calls, re-filing without operator review, automated re-dispute scheduling                                                |

## Live charge retry execution

Admin-gated live charge retry execution run audit from `retried` Stripe charge retry runs:

| Capability | Details                                                                                                                          |
| ---------- | -------------------------------------------------------------------------------------------------------------------------------- |
| Endpoints  | `GET /billing/live-charge-retry/status`, `GET .../runs`, `POST .../charge-retry-runs/{id}/execute`, `POST .../runs/{id}/approve` |
| Audit      | `stripe_live_charge_retry_execution_runs` table (migration `060`)                                                                |
| Gate       | `ENABLE_STRIPE_LIVE_CHARGE_RETRY_EXECUTION=true` (requires charge retry readiness)                                               |
| Deferred   | Live Stripe charge API calls without compliance deferral docs, automated collections without operator gates                      |

## HRIS passwordless UI
