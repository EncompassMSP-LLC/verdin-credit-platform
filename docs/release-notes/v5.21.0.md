# Release Notes — Version 5.21.0 (Reinvestigation Analytics & Evidence Polish)

**Version:** 5.21.0  
**Date:** 2026-07-16  
**Tag:** `v5.21.0`

## Summary

Version 5.21.0 advances **Compliance Intelligence Phase 14 — Reinvestigation Analytics & Evidence Polish**. It closes the documented tech debt left by 5.20 without touching the legal/compliance-gated frontier: outcome analytics gain a single-call per-recipient (bureau vs furnisher) breakdown, cross-bureau discrepancy detection compares `high_balance` and `credit_limit` with the existing $1.00 tolerance, and the litigation evidence PDF export uses a structured multi-section reportlab layout.

Everything remains staff-driven and advisory — the platform never polls a bureau, never files a dispute, never files suit, and never transmits the evidence export anywhere. Live bureau response ingestion, automated re-dispute filing, unsupervised escalation, automated litigation filing, and cross-tenant benchmarks remain explicitly out of scope. See [`docs/governance/version-5.21-scope.md`](../governance/version-5.21-scope.md).

## Shipped in v5.21.0

| Area                                         | Highlights                                                                                               |
| -------------------------------------------- | -------------------------------------------------------------------------------------------------------- |
| **Per-recipient analytics breakdown**        | Optional `group_by=recipient` returns a `by_recipient` roll-up (bureau vs furnisher vs unknown)          |
| **Cross-bureau high_balance / credit_limit** | New `high_balance_conflict` and `credit_limit_conflict` kinds in litigation-packet cross-bureau evidence |
| **Structured PDF litigation export**         | `format=pdf` uses reportlab platypus sections (headings, bullets, consistent typography)                 |
| **Governance**                               | Capability matrix 5.21 sign-off, scope doc, completion checklist, release notes                          |

## Per-recipient analytics breakdown

| Capability | Details                                                                                                                    |
| ---------- | -------------------------------------------------------------------------------------------------------------------------- |
| Endpoint   | `GET /reporting/reinvestigation-outcomes?group_by=recipient` (also accepts existing `start` / `end` / `bureau` filters)    |
| Payload    | `by_recipient: [{recipient, analytics}, …]` plus unchanged top-level `analytics` aggregate; `filters.group_by` echoed back |
| Guardrail  | Org-scoped only (no cross-tenant benchmarks); read-only; invalid `group_by` values return `422`                            |

## Cross-bureau high_balance / credit_limit

| Capability | Details                                                                                                          |
| ---------- | ---------------------------------------------------------------------------------------------------------------- |
| Logic      | Compare stored `high_balance` and `credit_limit` across sibling bureaus; differences ≤ $1.00 are rounding noise  |
| Payload    | New discrepancy kinds: `high_balance_conflict`, `credit_limit_conflict` (existing kinds unchanged)               |
| Guardrail  | Read model over stored tradeline data; no live bureau contact; still operator-gated behind the litigation packet |

## Structured PDF litigation evidence export

| Capability | Details                                                                                                                 |
| ---------- | ----------------------------------------------------------------------------------------------------------------------- |
| Endpoint   | `GET /accounts/{account_id}/litigation-packet/export?format=text\|pdf` (case_manager+ write; read-only users get `403`) |
| Payload    | Multi-section PDF with the same attorney-review content and disclaimer as the text export                               |
| Guardrail  | Manual handoff — the platform hands the operator a file; it never transmits it to a court/bureau/attorney or files      |

## Testing & CI

- Unit tests for `group_by=recipient` aggregation, high_balance/credit_limit conflict detection, tolerance bands, and structured PDF text extraction
- API tests for per-recipient analytics, cross-bureau monetary conflicts in litigation packets, and PDF attachment + write gate
- `@verdin/api-client` types + helpers for `group_by=recipient` and existing packet/analytics surfaces
- UI: Reporting Center "Break down by" selector (Bureau / Recipient) and per-recipient rates table

## Deferred to 5.22+ (or never)

- Live bureau response ingestion / polling (requires live bureau API access + legal/compliance sign-off) — 5.22+
- Automated re-dispute filing and unsupervised CFPB / attorney escalation (human gate retained) — 5.22+ / never
- Automated litigation filing / e-filing — **never** (the export is a human handoff; the platform never files)
- Cross-tenant reinvestigation-outcome benchmarks (data governance + legal review) — 5.22+
- Org-configurable cross-bureau balance tolerance — 5.22+
