# Release Notes — Version 5.19.0 (Reinvestigation Analytics & Evidence Depth)

**Version:** 5.19.0  
**Date:** 2026-07-15  
**Tag:** `v5.19.0`

## Summary

Version 5.19.0 advances **Compliance Intelligence Phase 12 — Reinvestigation Analytics & Evidence Depth**. It deepens the 5.18 reinvestigation and litigation foundations without touching the legal/compliance-gated frontier: outcome analytics gain date-range and per-bureau slicing, the FCRA §611 clock splits by recipient (credit bureau vs furnisher), the litigation-readiness packet folds in cross-bureau discrepancy evidence, and operators can export the packet as a text document for attorney handoff.

Everything remains staff-driven and advisory — the platform never polls a bureau, never files a dispute, never files suit, and never transmits the evidence export anywhere. Live bureau response ingestion, automated re-dispute filing, unsupervised escalation, automated litigation filing, and cross-tenant benchmarks remain explicitly out of scope. See [`docs/governance/version-5.19-scope.md`](../governance/version-5.19-scope.md).

## Shipped in v5.19.0

| Area                                          | Highlights                                                                                                          |
| --------------------------------------------- | ------------------------------------------------------------------------------------------------------------------- |
| **Reinvestigation analytics slicing**         | Optional `start` / `end` (by response day) + `bureau` filters on the per-org outcome analytics; filters echoed back |
| **Per-recipient §611 clock splits**           | Each clock entry carries a `recipients` array with an independent sub-clock per recipient (bureau vs furnisher)     |
| **Litigation packet cross-bureau evidence**   | Same-creditor tradelines at other bureaus compared; conflicts boost the willful-noncompliance grade                 |
| **Operator-gated litigation evidence export** | Downloadable `text/plain` litigation packet for a manual attorney handoff; never auto-transmitted                   |
| **Governance**                                | Capability matrix 5.19 sign-off, scope doc, completion checklist, release notes                                     |

## Reinvestigation analytics slicing

| Capability | Details                                                                                                          |
| ---------- | ---------------------------------------------------------------------------------------------------------------- |
| Endpoint   | `GET /reporting/reinvestigation-outcomes?start=&end=&bureau=`                                                    |
| Logic      | `bureau` filtered in SQL; `start`/`end` filtered in Python over the response day (`response_date`→`recorded_at`) |
| Payload    | Adds a `filters` block echoing the applied `start` / `end` / `bureau`                                            |
| Guardrail  | Still org-scoped only (no cross-tenant benchmarks); read-only aggregate; no live bureau contact                  |

## Per-recipient reinvestigation clock splits

| Capability | Details                                                                                                                                                                   |
| ---------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Payload    | `AccountReinvestigationClock.recipients[]` — per-recipient `clock_start_date`, `dispute_round_count`, `deadline`, `days_remaining`, `state`, `extended`, `response_count` |
| Logic      | Sent letters grouped by recipient; responses attributed to a recipient via the response's `dispute_letter_id`                                                             |
| Guardrail  | Read model only; top-level clock fields unchanged for single-recipient tradelines (backward compatible)                                                                   |

## Litigation packet cross-bureau evidence

| Capability | Details                                                                                                                                                                                                       |
| ---------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Payload    | `AccountLitigationPacket.cross_bureau` — `compared_bureaus` + typed `discrepancies` (`outcome_conflict`, `dispute_status_conflict`, `account_status_conflict`, `payment_status_conflict`, `balance_conflict`) |
| Logic      | Siblings matched on normalized creditor + masked account number and a different bureau; outcome conflicts add +25 to the score                                                                                |
| Guardrail  | Read model over stored data; no live bureau contact; still operator-gated behind `case_manager`+ write permission                                                                                             |

## Operator-gated litigation evidence export

| Capability | Details                                                                                                                                        |
| ---------- | ---------------------------------------------------------------------------------------------------------------------------------------------- |
| Endpoint   | `GET /accounts/{account_id}/litigation-packet/export?format=text` (case_manager+ write; read-only users get `403`)                             |
| Payload    | `text/plain` attachment with the tradeline summary, §611 clock, assessment + indicators, cross-bureau evidence, letters, responses, disclaimer |
| Guardrail  | Manual handoff — the platform hands the operator a file; it never transmits it to a court/bureau/attorney or files                             |

## Testing & CI

- Unit tests for the cross-bureau detector, litigation-readiness cross-bureau folding, and the text export formatter
- API tests for analytics date-range/bureau filters, per-recipient clock splits + selective response attribution, cross-bureau conflict surfacing, and the operator-gated export (incl. write-permission gate)
- `@verdin/api-client` types + helpers for the sliced analytics, recipient sub-clocks, cross-bureau evidence, and packet download
- New UI: analytics from/to + bureau filters, per-recipient clock sub-rows, cross-bureau discrepancy section, and a "Download evidence (.txt)" button

## Deferred to 5.20+ (or never)

- Live bureau response ingestion / polling (requires live bureau API access + legal/compliance sign-off) — 5.20+
- Automated re-dispute filing and unsupervised CFPB / attorney escalation (human gate retained) — 5.20+ / never
- Automated litigation filing / e-filing — **never** (the export is a human handoff; the platform never files)
- Cross-tenant reinvestigation-outcome benchmarks (data governance + legal review) — 5.20+
