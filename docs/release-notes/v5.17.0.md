## Summary

Version 5.17.0 advances **Compliance Intelligence Phase 10 ŌĆö Dispute Response & Reinvestigation Tracking**. It closes the dispute loop: operators record auditable bureau/furnisher responses to sent disputes, the platform computes the FCRA ┬¦611 reinvestigation clock, surfaces advisory re-dispute / escalation readiness, and rolls everything up into a per-case reinvestigation dashboard.

Every response is staff-entered ŌĆö the platform never polls a bureau ŌĆö and every recommendation is advisory. Live bureau response ingestion, automated re-dispute filing, and unsupervised escalation remain explicitly out of scope. See [`docs/governance/version-5.17-scope.md`](../governance/version-5.17-scope.md).

## Dispute response intake & persistence

| Capability  | Details                                                                                                                          |
| ----------- | -------------------------------------------------------------------------------------------------------------------------------- |
| Endpoints   | `POST /accounts/{id}/dispute-responses`, `GET /accounts/{id}/dispute-responses`                                                  |
| Persistence | Migration `087_dispute_responses` ŌĆö outcome, method, response date, notes, optional links to a sent letter and a case document   |
| Behavior    | Terminal outcomes sync `dispute_status`/`response_received`/investigation status + emit a timeline event; `no_response` does not |
| Guardrail   | Staff-entered only; the platform never contacts a bureau                                                                         |

## ┬¦611 reinvestigation clock

| Capability | Details                                                                                                          |
| ---------- | ---------------------------------------------------------------------------------------------------------------- |
| Endpoint   | `GET /accounts/reinvestigation-clock?case_id=`                                                                   |
| Logic      | 30-day window from `last_dispute_date`; states `not_sent`/`awaiting`/`due_soon` (Ōēż7d)/`overdue`/`responded`      |
| Guardrail  | Pure read model over stored dates + responses; `no_response` never counts as `responded`; no live bureau contact |

## Re-dispute / escalation readiness

| Capability | Details                                                                                                                     |
| ---------- | --------------------------------------------------------------------------------------------------------------------------- |
| Endpoint   | `GET /accounts/redispute-readiness?case_id=`                                                                                |
| Logic      | `overdue` ŌåÆ re-dispute; `verified` ŌåÆ MOV re-dispute, CFPB (multi-round), or attorney (`risk_score Ōēź 80`); `rejected` ŌåÆ CFPB |
| Guardrail  | Advisory only ŌĆö never files a dispute or escalation; `latest_outcome` falls back to the account's terminal `dispute_status` |

## Case reinvestigation dashboard

| Capability | Details                                                                                                                    |
| ---------- | -------------------------------------------------------------------------------------------------------------------------- |
| Endpoint   | `GET /accounts/reinvestigation-summary?case_id=`                                                                           |
| Payload    | Totals, per-state clock summary, per-action readiness summary, `next_deadline`, `most_overdue_days`, high-priority actions |
| Guardrail  | Aggregation over the clock/readiness read models ŌĆö no writes, no bureau contact                                            |

## Testing & CI

## Deferred to 5.18+

- Live bureau response ingestion (requires live bureau API access + legal/compliance sign-off)
- Automated re-dispute filing and unsupervised escalation (human gate retained)
- Extended 45-day reinvestigation window modeling and per-letter (`sent_at`) multi-round clocks
