# Release Notes — Version 18.0.0 (Reinvestigation Operations Polish)

**Version:** 18.0.0  
**Date:** 2026-07-16  
**Tag:** `v18.0.0`

## Summary

Version 18.0.0 advances **Compliance Intelligence Phase 17 — Reinvestigation Operations Polish**. It closes the operator polish gaps left by 16.0/17.0: organizations can configure default baseline/recent windows for org-internal reinvestigation outcome benchmarks, and Compliance Center ingestion audit runs can be scoped to an optional case or account from the UI.

Everything remains staff-driven and advisory — the platform never polls a bureau, never files a dispute, never escalates without a human, never files suit, and never shares reinvestigation-outcome data across tenants. Live bureau response ingestion execution, automated re-dispute filing execution, unsupervised escalation, automated litigation filing, and cross-tenant benchmarks remain explicitly out of scope. See [`docs/governance/version-18.0-scope.md`](../governance/version-18.0-scope.md).

## Org-configurable benchmark window defaults

| Capability | Details                                                                          |
| ---------- | -------------------------------------------------------------------------------- |
| Surface    | Org Admin → Dispute settings; Reporting Center → Outcome benchmarks              |
| API        | `GET`/`PATCH /org-admin/dispute-settings`; benchmarks omit days → org defaults   |
| Guardrail  | Org-scoped only; recent ≤ baseline; no cross-tenant data; no live bureau contact |

## Ingestion audit case/account scope UI

| Capability | Details                                                                                         |
| ---------- | ----------------------------------------------------------------------------------------------- |
| Surface    | Compliance center → **Response ingestion**                                                      |
| API        | Existing `case_id` / `account_id` on list + start under `/compliance/bureau-response-ingestion` |
| Guardrail  | Start always records `status=deferred`; UUID validation client-side; no external bureau calls   |
