# Release Notes — Version 26.0.0 (Document Pipeline Resolution & Operator Surfaces)

**Version:** 26.0.0  
**Date:** 2026-07-21  
**Tag:** `v26.0.0`

## Summary

Version 26.0.0 advances **Compliance Intelligence Phase 25 — Document Pipeline Resolution & Operator Surfaces**. It closes remaining non-blocked document-pipeline **operator surface** gaps: Case Detail hosts bulk recovery actions whenever any case documents exist (not gated on classified credit reports), and staff can enqueue async entity re-resolve when metadata is already extracted.

Everything remains staff-driven and advisory — the platform never polls a bureau, never files a dispute, never escalates without a human, never files suit, and never shares reinvestigation-outcome data across tenants. Live bureau response ingestion execution, automated re-dispute filing execution, unsupervised escalation, automated litigation filing, and cross-tenant benchmarks remain explicitly out of scope. See [`docs/governance/version-26.0-scope.md`](../governance/version-26.0-scope.md).

## Case Documents recovery panel

| Capability | Details                                                                                                  |
| ---------- | -------------------------------------------------------------------------------------------------------- |
| Surface    | Case Detail → Case Documents recovery                                                                    |
| APIs       | Existing case bulk endpoints (`ocr/retry`, `classify/reclassify`, `metadata/reextract`, parsed re-parse) |
| Guardrail  | Visible when any case documents exist; not gated on `document_type=credit_report`; staff-mediated only   |

## Operator async entity re-resolve enqueue

| Capability | Details                                                                                                          |
| ---------- | ---------------------------------------------------------------------------------------------------------------- |
| Surface    | Document Detail → Entity matches → Re-resolve (async)                                                            |
| API        | `POST /documents/{id}/resolutions/reresolve`                                                                     |
| Guardrail  | Requires extracted metadata (422 otherwise); 503 when entity resolution disabled; sync Resolve remains available |
