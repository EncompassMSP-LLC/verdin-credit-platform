# Release Notes — Version 5.16.0 (Identity-Theft Recovery Depth & §605B Evidence Bundling)

**Version:** 5.16.0  
**Date:** 2026-07-15  
**Tag:** `v5.16.0`

## Summary

Version 5.16.0 advances **Compliance Intelligence Phase 9 — Identity-Theft Recovery Depth & §605B Evidence Bundling**. It builds on the Phase 8 Identity Theft Case Center by letting operators bundle staff-selected evidence into the §605B packet, surfacing advisory mixed-file / personal-info variation signals, recording operator-gated §605B submission-readiness audits, and making bulk dispute preparation identity-theft-lock-aware.

Live unsupervised §605B bureau submission and unsupervised sworn claim generation remain explicitly out of scope. See [`docs/governance/version-5.16-scope.md`](../governance/version-5.16-scope.md).

## Shipped in v5.16.0

| Area                                   | Highlights                                                                                                      |
| -------------------------------------- | --------------------------------------------------------------------------------------------------------------- |
| **§605B evidence exhibit bundling**    | Staff-selected `document_id` exhibits bundled into the packet `exhibits/` folder; type/size gated; no auto-send |
| **Mixed-file / personal-info signals** | Advisory `PERSONAL_INFO` variation findings (multiple SSN/DOB/name/address); never auto-labels an account       |
| **§605B submission-readiness audit**   | Operator-gated readiness run persisted to an audit table with per-bureau coverage + blocking reasons            |
| **Lock-aware dispute preparation**     | Bulk prepare skips identity-theft-locked tradelines into `locked` instead of aborting the batch with a `409`    |
| **Governance**                         | Capability matrix 5.16 sign-off, scope doc, completion checklist, release notes                                 |

## §605B evidence exhibit bundling

| Capability | Details                                                                                                            |
| ---------- | ------------------------------------------------------------------------------------------------------------------ |
| Endpoint   | `GET /cases/{id}/identity-theft/605b-packet.zip` with repeated `document_id` query params                          |
| Gating     | MIME type (PDF/images/plain text), per-file size (15 MB), total size (40 MB); skipped/missing recorded in manifest |
| Guardrail  | Nothing auto-attached; documents outside the case are rejected; export never fails on a bad exhibit                |

## Mixed-file / personal-info variation detection

| Capability | Details                                                                                                                        |
| ---------- | ------------------------------------------------------------------------------------------------------------------------------ |
| Signals    | Multiple unmasked SSNs, multiple DOBs, distinct surname variations, many address variations (`identity_theft.personal_info.*`) |
| Source     | New `PERSONAL_INFO` detection source; counted in `summary.personal_info_indicators`; mixed-file badge in Case Center           |
| Guardrail  | Advisory only — does not lock ordinary §611 disputes, does not trip the fraud-alert banner, never auto-labels theft            |

## §605B submission-readiness audit

| Capability  | Details                                                                                                                   |
| ----------- | ------------------------------------------------------------------------------------------------------------------------- |
| Endpoints   | `POST /cases/{id}/identity-theft/605b-readiness-runs`, `GET .../605b-readiness-runs/latest`                               |
| Persistence | Migration `086_it_readiness_runs` — confirmed count, attestation, per-bureau coverage, missing evidence, blocking reasons |
| Guardrail   | Operator-gated audit trail only; never submits to a bureau                                                                |

## Lock-aware dispute preparation

| Capability | Details                                                                                                       |
| ---------- | ------------------------------------------------------------------------------------------------------------- |
| Endpoints  | `POST /cases/{id}/dispute-strategy/prepare`, `POST /cases/{id}/credit-report-discrepancies/prepare-disputes`  |
| Behavior   | Identity-theft-locked tradelines are skipped into a `locked` array (`match_key`, `creditor_name`, `reason`)   |
| Guardrail  | No account is created for a locked tradeline; operators are steered to the Identity Theft Case Center (§605B) |

## Testing & CI

- Unit tests for exhibit gating, personal-info variation rules, and §605B packet builders
- API tests for readiness-audit runs, evidence bundling export, and lock-aware prepare (strategy + cross-bureau paths)
- `@verdin/api-client` types + helpers for readiness runs, exhibits, and `locked` prepare results

## Deferred to 5.17+

- Live unsupervised §605B bureau submission (legal/compliance sign-off + kill-switch required)
- Unsupervised sworn identity-theft claim generation (consumer confirmation stays a human gate)
- Persisted exhibit selection sets and unrestricted cross-tenant PII export
