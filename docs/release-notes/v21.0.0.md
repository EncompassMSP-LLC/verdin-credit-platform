# Release Notes — Version 21.0.0 (Reinvestigation Operations Filters)

**Version:** 21.0.0  
**Date:** 2026-07-17  
**Tag:** `v21.0.0`

## Summary

Version 21.0.0 advances **Compliance Intelligence Phase 20 — Reinvestigation Operations Filters**. It closes remaining non-blocked configuration and audit-filter gaps: organizations can set optional per-recipient (credit bureau vs furnisher) benchmark window overrides on dispute settings, and Compliance Center ingestion audit run lists filter by `bureau_target` and `status`.

Everything remains staff-driven and advisory — the platform never polls a bureau, never files a dispute, never escalates without a human, never files suit, and never shares reinvestigation-outcome data across tenants. Live bureau response ingestion execution, automated re-dispute filing execution, unsupervised escalation, automated litigation filing, and cross-tenant benchmarks remain explicitly out of scope. See [`docs/governance/version-21.0-scope.md`](../governance/version-21.0-scope.md).

## Per-recipient benchmark window defaults

| Capability | Details                                                                                                            |
| ---------- | ------------------------------------------------------------------------------------------------------------------ |
| Surface    | Org Admin → Dispute settings → Per-recipient window overrides; Reporting Center optional Recipient filter          |
| API        | `reinvestigation_benchmark_recipient_windows` on dispute settings; optional `recipient` on benchmarks + CSV export |
| Guardrail  | Org-scoped only; invalid recipient keys → 422; recent ≤ baseline; null clears an override                          |

## Ingestion audit bureau/status list filters

| Capability | Details                                                                    |
| ---------- | -------------------------------------------------------------------------- |
| Surface    | Compliance Center → Response ingestion → Filter bureau / Filter status     |
| API        | `GET /compliance/bureau-response-ingestion/runs?bureau_target=&status=`    |
| Guardrail  | Invalid `status` → 422; start-run remains always `deferred` (no live poll) |
