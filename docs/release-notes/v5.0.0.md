# Release Notes — Version 5.0.0 (Enterprise Edition)

**Version:** 5.0.0  
**Date:** 2026-07-02  
**Tag:** `v5.0.0`

## Summary

Version 5.0.0 is the **Enterprise Edition** release. It extends the Operations platform (v4.8.0) with durable case–client linking, production email delivery, post-gate LLM case summaries, job orchestrator wiring, enterprise identity readiness, compliance center foundations, expanded client portal capabilities, enterprise reporting read models, and organization admin with API key lifecycle — all behind feature flags and documented **Partial** scope limits.

Nine epics ship as **Partial** with explicit deferrals to 5.0+ / 5.1+ for autonomous agents, full BPM, billing, predictive analytics, and native mobile. See [`docs/governance/version-5.0-scope.md`](../governance/version-5.0-scope.md).

## Shipped in v5.0.0

| Area                     | Highlights                                                                                         |
| ------------------------ | -------------------------------------------------------------------------------------------------- |
| **Client linking**       | `cases.client_id` FK; staff API updates; portal case matching prefers FK when set                  |
| **Production email**     | SMTP/SendGrid adapters, delivery audit log, `POST /notifications/email/send`, workflow integration |
| **LLM case summaries**   | `POST /cases/{id}/llm-summary` behind `ENABLE_LLM` + ADR-012; PII scrubbing; timeline audit        |
| **Job orchestrator**     | Runner retry/metrics wiring; croniter scheduler; `overdue_investigation_scan` at `0 6 * * *` UTC   |
| **Enterprise identity**  | `GET /enterprise/status` SSO (`oidc`/`saml`) and MFA (`totp`) readiness behind `ENABLE_ENTERPRISE` |
| **Compliance center**    | Consent records + withdrawal; retention policy placeholders; `GET /compliance/status`              |
| **Portal expansion**     | Client document upload on linked cases; secure messaging thread scaffold (portal + staff)          |
| **Enterprise reporting** | Bureau performance + team productivity read models; `GET /reporting/status`                        |
| **Org admin**            | Organization summary, API key create/list/revoke with hashed storage (`vrd_live_` prefix)          |
| **Governance**           | Capability matrix 5.0 sign-off, scope doc, completion checklist, release notes                     |

## Data & client linking

- Alembic migration adds optional `cases.client_id` foreign key to `clients`
- Staff case create/update accepts `client_id`; portal case list/detail prefer FK match over email/name heuristics
- Bulk import, CRM sync, and full intake wizard deferred to 5.0+

## Communications

Production email delivery ships behind `ENABLE_EMAIL_DELIVERY`:

| Capability | Endpoints / behavior                                               |
| ---------- | ------------------------------------------------------------------ |
| Send       | `POST /notifications/email/send`                                   |
| Audit      | `email_delivery_logs` table; `GET /notifications/email/deliveries` |
| Providers  | `smtp` and `sendgrid` adapters                                     |
| Workflow   | Optional `deliver_email` on notification create                    |

SMS production delivery, marketing campaigns, and deliverability dashboards remain deferred.

## LLM case summaries

Post-gate LLM assistance for staff case managers:

- `POST /cases/{case_id}/llm-summary` — external provider call only when `require_llm_ready()` passes
- `verdin_llm_gateway` completion clients (OpenAI-compatible, Anthropic)
- Scrubbed context via `scrub_payload()`; timeline event `CASE_LLM_SUMMARY_GENERATED` with model + prompt hash metadata
- `GET /llm/status` unchanged from 4.8 — readiness gate before any provider call

Document summaries, LLM dispute augmentation, and autonomous agents → **5.0+** / **5.1+**.

## Platform operations

- `JobScheduler` cron evaluation (croniter) in worker orchestrator
- `RetryPolicy` and `JobMetricsRecorder` wired into `worker/orchestrator.py`
- `overdue_investigation_scan` registered at `0 6 * * *` UTC with in-process scheduler ticks
- PostgreSQL job persistence, cron admin UI, and cross-org SLA dashboards deferred

## Enterprise identity

MFA and SSO readiness scaffold for staff users (portal auth partition unchanged):

| Capability | Details                                                         |
| ---------- | --------------------------------------------------------------- |
| Status     | `GET /enterprise/status`                                        |
| SSO        | `ENTERPRISE_SSO_PROVIDER` (`oidc` / `saml`) + issuer/client env |
| MFA        | `ENTERPRISE_MFA_MODE=totp` + `ENTERPRISE_MFA_ISSUER`            |
| Gate       | `ENABLE_ENTERPRISE=true` required                               |

No external IdP or TOTP enrollment calls execute in this release. SCIM provisioning and per-org SAML metadata UI → **5.0+**.

## Compliance center

Org-scoped compliance foundations for CROA/FCRA-oriented operations:

| Capability         | Endpoints                                                      |
| ------------------ | -------------------------------------------------------------- |
| Status             | `GET /compliance/status`                                       |
| Consent records    | `GET/POST /compliance/consents`, withdraw                      |
| Retention policies | `GET/POST/PATCH /compliance/retention-policies` (admin create) |

Consent types: `croa_services`, `fcra_dispute`, `fdcpa_contact`, `marketing`, `data_processing`. Enforcement jobs and legal sign-off workflows deferred.

## Client portal expansion

Enable with `ENABLE_CLIENT_PORTAL=true` (API) and `VITE_ENABLE_CLIENT_PORTAL=true` (web).

| Capability       | Endpoints / behavior                                    |
| ---------------- | ------------------------------------------------------- |
| Document upload  | `POST/GET /portal/cases/{case_id}/documents`            |
| Secure messaging | `GET/POST /portal/cases/{case_id}/messages`             |
| Staff messaging  | `GET/POST /cases/{case_id}/message-thread`              |
| Timeline         | `PORTAL_DOCUMENT_UPLOADED`, portal/staff message events |

Billing, credit score history charts, mobile push, and real-time message delivery deferred.

## Enterprise reporting

Org-scoped aggregate read models for enterprise dashboards:

| Endpoint                            | Description                                          |
| ----------------------------------- | ---------------------------------------------------- |
| `GET /reporting/status`             | Enterprise reporting capabilities overview           |
| `GET /reporting/bureau-performance` | Tradeline counts and dispute outcomes by bureau      |
| `GET /reporting/team-productivity`  | Per-staff task and case productivity (30-day window) |

`GET /reporting/operations` from 4.8 remains unchanged. Materialized views and revenue metrics deferred.

## Organization admin

Behind `ENABLE_ENTERPRISE=true`; **admin** role required:

| Endpoint                               | Description                                    |
| -------------------------------------- | ---------------------------------------------- |
| `GET /org-admin/status`                | Org admin capabilities overview                |
| `GET /org-admin/organization`          | Organization summary (users, active API keys)  |
| `GET/POST /org-admin/api-keys`         | List / create API keys (full secret on create) |
| `GET /org-admin/api-keys/{id}`         | API key metadata                               |
| `POST /org-admin/api-keys/{id}/revoke` | Revoke active key                              |

API keys use `vrd_live_` prefix with SHA-256 hashed storage. Scopes: `read`, `write`. Key-authenticated API middleware, SCIM, and billing admin deferred.
