# Release Notes — Version 4.5.0 (Automation Platform)

**Version:** 4.5.0  
**Date:** 2026-06-30  
**Tag:** `v4.5.0`

## Summary

Version 4.5.0 is the **Automation Platform** release. It builds on the Operational Core (v4.3.x) with credit report import intelligence, event-driven workflow tasks, and a staff-mediated dispute letter lifecycle — without modifying foundational CRUD contracts.

Three epics ship as **Partial** with documented scope limits (workflow auto-tasks, dispute foundation, rules-based AI). Client experience and LLM assistance are **deferred to 4.8**. See [`docs/governance/version-4.5-scope.md`](../governance/version-4.5-scope.md).

## Credit Report Intelligence

Staff can import bureau PDFs and follow the document pipeline from one screen:

- **Import wizard** (`/imports/credit-report`) — upload, OCR/classification tracking, metadata extraction, entity resolution, in-wizard OCR retry and re-extraction
- **Bureau parsers** — structured `ParsedCreditReport` output for Equifax, Experian, and TransUnion layouts via `packages/report-parsers/`
- **Tradeline review** — parsed report provenance, account candidates, review tasks (manual + worker auto-create)
- **Historical comparison** — case and account detail views compare current vs previous bureau report tradeline deltas
- **Duplicate detection** — SHA-256 hash matching surfaces canonical prior imports in wizard, documents list, and case history

Enable with `VITE_ENABLE_IMPORTS=true`.

## Dispute Letter Lifecycle

Rule-based dispute letters tie into account intelligence and tasks:

| Stage    | Capability                                                                 |
| -------- | -------------------------------------------------------------------------- |
| Preview  | `GET /accounts/{id}/dispute-draft` — CRA or furnisher template             |
| Persist  | `POST /accounts/{id}/dispute-draft/letters`                                |
| Review   | Auto review task; approve transitions letter to `approved`                 |
| Send     | Mark sent; sync account `dispute_status`; auto-create “Track CRA response” |
| Awaiting | `POST .../dispute-awaiting-response`                                       |
| Outcome  | `POST .../dispute-response-received` (verified / corrected / deleted)      |
| Overdue  | Auto-escalation task when CRA window passes                                |
| Export   | `GET .../dispute-letters/{id}/export?format=text\|pdf`                     |
| Void     | Cancel in-flight letters in draft/review/approved                          |

UI: dispute draft preview, saved letter panel with detail expand, export buttons, response outcome actions on account detail.

## Workflow Auto-Tasks

Event-driven task creation (not a full BPM engine):

- Dispute draft review (`accounts.dispute_draft`)
- Dispute letter review (`accounts.dispute_letter`)
- CRA follow-up on send (`accounts.dispute_letter_followup`, 30-day due)
- Overdue investigation escalation (`accounts.dispute_investigation_overdue`)
- Parsed credit report review (`documents.parsed_credit_report`) — worker auto-create

Tasks list supports filtering by `source_module`.
