# Release Notes — Version 5.20.0 (Reinvestigation Analytics & Evidence Refinement)

**Version:** 5.20.0  
**Date:** 2026-07-16  
**Tag:** `v5.20.0`

## Summary

Version 5.20.0 advances **Compliance Intelligence Phase 13 — Reinvestigation Analytics & Evidence Refinement**. It closes the documented tech debt left by 5.19 without touching the legal/compliance-gated frontier: outcome analytics gain a single-call per-bureau breakdown, the FCRA §611(a)(1)(B) extended-window flag is computed per recipient sub-clock, the litigation evidence export adds a PDF format, and cross-bureau discrepancy detection gains a $1.00 monetary tolerance plus past-due / date-reported field comparisons.

Everything remains staff-driven and advisory — the platform never polls a bureau, never files a dispute, never files suit, and never transmits the evidence export anywhere. Live bureau response ingestion, automated re-dispute filing, unsupervised escalation, automated litigation filing, and cross-tenant benchmarks remain explicitly out of scope. See [`docs/governance/version-5.20-scope.md`](../governance/version-5.20-scope.md).

## Shipped in v5.20.0

| Area                                       | Highlights                                                                                            |
| ------------------------------------------ | ----------------------------------------------------------------------------------------------------- |
| **Per-bureau analytics breakdown**         | Optional `group_by=bureau` returns a `by_bureau` roll-up so operators compare all bureaus in one call |
| **Per-recipient extended-window accuracy** | §611(a)(1)(B) 45-day flag evaluated against each recipient's own `clock_start_date`                   |
| **PDF litigation evidence export**         | `format=pdf` on the operator-gated litigation packet export (reportlab); text remains the default     |
| **Cross-bureau discrepancy depth**         | $1.00 balance/past-due tolerance; new `past_due_conflict` and `date_reported_conflict` kinds          |
| **Governance**                             | Capability matrix 5.20 sign-off, scope doc, completion checklist, release notes                       |

## Per-bureau analytics breakdown

| Capability | Details                                                                                                              |
| ---------- | -------------------------------------------------------------------------------------------------------------------- |
| Endpoint   | `GET /reporting/reinvestigation-outcomes?group_by=bureau` (also accepts existing `start` / `end` / `bureau` filters) |
| Payload    | `by_bureau: [{bureau, analytics}, …]` plus unchanged top-level `analytics` aggregate; `filters.group_by` echoed back |
| Guardrail  | Org-scoped only (no cross-tenant benchmarks); read-only; invalid `group_by` values return `422`                      |

## Per-recipient extended-window accuracy

| Capability | Details                                                                                                                          |
| ---------- | -------------------------------------------------------------------------------------------------------------------------------- |
| Payload    | Each `recipients[]` sub-clock's `extended` flag is independent of the tradeline-level flag                                       |
| Logic      | `document_extends_window` runs against that recipient's `clock_start_date` and the shared account/case document dates            |
| Guardrail  | Read model only; no new document collection; a document in an early bureau window no longer stretches a later furnisher deadline |

## PDF litigation evidence export

| Capability | Details                                                                                                                 |
| ---------- | ----------------------------------------------------------------------------------------------------------------------- |
| Endpoint   | `GET /accounts/{account_id}/litigation-packet/export?format=text\|pdf` (case_manager+ write; read-only users get `403`) |
| Payload    | `application/pdf` or `text/plain` attachment with the same attorney-review content and disclaimer                       |
| Guardrail  | Manual handoff — the platform hands the operator a file; it never transmits it to a court/bureau/attorney or files      |

## Cross-bureau discrepancy depth

| Capability | Details                                                                                                                         |
| ---------- | ------------------------------------------------------------------------------------------------------------------------------- |
| Logic      | Balance and past-due differences ≤ $1.00 are treated as rounding noise; `past_due_amount` and `date_reported` are also compared |
| Payload    | New discrepancy kinds: `past_due_conflict`, `date_reported_conflict` (existing kinds unchanged)                                 |
| Guardrail  | Read model over stored parsed-report data; no live bureau contact; still operator-gated behind the litigation packet            |

## Testing & CI

- Unit tests for per-bureau `group_by` aggregation, per-recipient extended-window isolation, PDF export bytes, balance tolerance, and new field conflicts
- API tests for analytics breakdown, recipient extension, PDF attachment + write gate, and cross-bureau tolerance
- `@verdin/api-client` types + helpers for `group_by`, PDF download format, and existing packet/analytics surfaces
- UI: per-bureau rates table, per-recipient §611(a)(1)(B) badge, Download .txt / .pdf buttons

## Deferred to 5.21+ (or never)

- Live bureau response ingestion / polling (requires live bureau API access + legal/compliance sign-off) — 5.21+
- Automated re-dispute filing and unsupervised CFPB / attorney escalation (human gate retained) — 5.21+ / never
- Automated litigation filing / e-filing — **never** (the export is a human handoff; the platform never files)
- Cross-tenant reinvestigation-outcome benchmarks (data governance + legal review) — 5.21+
