# Release Notes — Version 22.0.0 (Document Pipeline Hardening)

**Version:** 22.0.0  
**Date:** 2026-07-21  
**Tag:** `v22.0.0`

## Summary

Version 22.0.0 advances **Compliance Intelligence Phase 21 — Document Pipeline Hardening**. It closes non-blocked document-pipeline gaps found in pilot operations: `document_metadata.payment_status` accepts long bureau status narratives (varchar 255), and staff can re-enqueue credit report parse for OCR'd, classified credit reports from Document Detail.

Everything remains staff-driven and advisory — the platform never polls a bureau, never files a dispute, never escalates without a human, never files suit, and never shares reinvestigation-outcome data across tenants. Live bureau response ingestion execution, automated re-dispute filing execution, unsupervised escalation, automated litigation filing, and cross-tenant benchmarks remain explicitly out of scope. See [`docs/governance/version-22.0-scope.md`](../governance/version-22.0-scope.md).

## Widen metadata payment_status

| Capability | Details                                                        |
| ---------- | -------------------------------------------------------------- |
| Surface    | Schema + worker metadata table (no UI change)                  |
| Migration  | `093_widen_meta_pay_status`                                    |
| Guardrail  | Schema-only widen; no enum normalization; no PII export change |

## Operator re-parse credit report

| Capability | Details                                                                                       |
| ---------- | --------------------------------------------------------------------------------------------- |
| Surface    | Document Detail → Re-parse credit report (when OCR present and type is credit_report)         |
| API        | `POST /documents/{id}/parsed-credit-report/reparse`                                           |
| Guardrail  | Org-scoped; 422 without OCR or non-credit_report; case_manager+; no live bureau; no bulk/auto |
