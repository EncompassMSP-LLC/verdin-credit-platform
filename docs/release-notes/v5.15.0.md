# Release Notes — Version 5.15.0 (Identity Theft Detection & Recovery)

**Version:** 5.15.0  
**Date:** 2026-07-14  
**Tag:** `v5.15.0`

## Summary

Version 5.15.0 completes **Compliance Intelligence Phase 8 — Identity Theft Detection & Recovery**. It ships report and tradeline identity-theft indicators, consumer confirmation + attestation gates (staff and portal), Identity Theft Case Center persistence, ordinary dispute pause for locked accounts, and staff-mediated FCRA §605B block packet export.

Live unsupervised §605B bureau submission and unsupervised sworn claim generation remain explicitly out of scope. See [`docs/governance/version-5.15-scope.md`](../governance/version-5.15-scope.md).

## Shipped in v5.15.0

| Area                                    | Highlights                                                                                          |
| --------------------------------------- | --------------------------------------------------------------------------------------------------- |
| **Identity Theft Detection & Recovery** | Phase 8 engine, Case Center (incident/reviews/protections), staff APIs + Case Detail UI             |
| **Portal confirmation**                 | Portal Identity Theft Center read + tradeline confirmation with attestation                         |
| **§605B block packet (Partial)**        | Staff ZIP export with per-bureau block letters + readiness manifest (`409` without confirmed theft) |
| **Dispute pause**                       | Ordinary dispute letter drafts return `409` while indicator or confirmed claim locks an account     |
| **Governance**                          | Capability matrix 5.15 sign-off, scope doc, completion checklist, release notes                     |

## Identity Theft Detection & Recovery Engine

| Capability  | Details                                                                                                                             |
| ----------- | ----------------------------------------------------------------------------------------------------------------------------------- |
| Detection   | Report phrases (fraud alert, freeze, victim statement) and tradeline heuristics; findings are investigator aids                     |
| Persistence | Migration `085_identity_theft` — incidents, account reviews, protections                                                            |
| Endpoints   | `GET /cases/{id}/identity-theft-findings`, `GET .../identity-theft-center`, `POST .../account-reviews`, protections/incident writes |
| Gate        | Confirmation choice `identity_theft` requires `attestation_accepted`; never auto-labels an account as identity theft                |

## Portal consumer confirmation

| Capability | Details                                                                                                  |
| ---------- | -------------------------------------------------------------------------------------------------------- |
| Endpoints  | `GET /portal/cases/{id}/identity-theft-center`, `POST /portal/cases/{id}/identity-theft/account-reviews` |
| UI         | Portal case detail — review flagged tradelines and attest before confirmed claim                         |
| Scope      | Portal JWT + client case access only; no staff protection/incident admin APIs                            |

## FCRA §605B block packet (Partial)

| Capability | Details                                                                              |
| ---------- | ------------------------------------------------------------------------------------ |
| Endpoint   | `GET /cases/{id}/identity-theft/605b-packet.zip?letter_format=pdf\|text`             |
| Contents   | `README.md` readiness manifest + `letters/` per-bureau §605B block request letters   |
| Gate       | Requires consumer-confirmed identity-theft review with attestation (`409` otherwise) |
| Deferred   | Live bureau submission; auto-bundled evidence exhibits                               |
