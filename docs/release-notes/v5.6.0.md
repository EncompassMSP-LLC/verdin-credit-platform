# Release Notes — Version 5.6.0 (Compliance-Reviewed Production Depth)

**Version:** 5.6.0  
**Date:** 2026-07-05  
**Tag:** `v5.6.0`

## Summary

Version 5.6.0 completes the **Compliance-Reviewed Production Depth** milestone after v5.5.0. It ships HRIS bidirectional sync scaffold, SMS deliverability dashboards, ADR-012-gated LLM dispute draft augmentation, and compliance-gated dispute filing prep — all behind feature flags and documented **Partial** scope limits.

All four 5.6 epics ship as **Partial ✅**. See [`docs/governance/version-5.6-scope.md`](../governance/version-5.6-scope.md).

## Shipped in v5.6.0

| Area                    | Highlights                                                                             |
| ----------------------- | -------------------------------------------------------------------------------------- |
| **HRIS sync**           | Bidirectional sync run audit + status scaffold behind `ENABLE_HRIS_BIDIRECTIONAL_SYNC` |
| **SMS deliverability**  | Marketing SMS delivery metrics read model + staff dashboard endpoints                  |
| **LLM dispute augment** | ADR-012-gated dispute draft augment audit + timeline correlation (no auto-send)        |
| **Dispute filing prep** | Compliance-gated filing prep run audit with admin approval (no bureau submission)      |
| **Governance**          | Capability matrix 5.6 sign-off, scope doc, completion checklist, release notes         |

## HRIS bidirectional sync

Org-scoped HRIS sync run audit aligned with enterprise federation scaffold:

| Capability | Details                                                                                       |
| ---------- | --------------------------------------------------------------------------------------------- |
| Endpoints  | `GET /enterprise/federation/hris-sync/status`, `GET .../runs`, `POST .../run`                 |
| Audit      | `hris_bidirectional_sync_runs` table (migration `040`)                                        |
| Gate       | `ENABLE_HRIS_BIDIRECTIONAL_SYNC=true` (requires `ENABLE_ENTERPRISE` + SAML metadata scaffold) |
| Deferred   | Full employee lifecycle sync, passwordless enrollment UI, automated certificate rotation      |

## SMS deliverability dashboard

Delivery metrics aggregate read model for marketing SMS campaigns:

| Capability | Details                                                                                    |
| ---------- | ------------------------------------------------------------------------------------------ |
| Endpoints  | `GET /notifications/sms-campaigns/deliverability/status`, `GET .../deliverability/summary` |
| Gate       | `ENABLE_SMS_DELIVERABILITY_DASHBOARD=true` (requires `ENABLE_SMS_MARKETING_DELIVERY`)      |
| Deferred   | Multi-provider failover, A/B testing, opt-out compliance automation, real-time alerting    |

## LLM dispute draft augment

Staff-initiated LLM augment for dispute letter drafts with PII scrubbing and audit trail:

| Capability | Details                                                                                                  |
| ---------- | -------------------------------------------------------------------------------------------------------- |
| Endpoints  | `GET /llm/dispute-draft/status`, `GET /accounts/{id}/dispute-draft/llm-augments`, `POST .../llm-augment` |
| Audit      | `llm_dispute_draft_augments` table (migration `041`)                                                     |
| Timeline   | Case-scoped augments emit `DISPUTE_DRAFT_LLM_AUGMENT` platform events                                    |
| Gate       | `ENABLE_LLM_DISPUTE_DRAFT_AUGMENT=true` (requires `ENABLE_LLM` + ADR-012 scrub config)                   |
| Deferred   | Autonomous letter send, unsupervised LLM loops, PII export without scrub config                          |

## Dispute filing prep (compliance-gated)

Staff-submitted filing prep runs requiring admin approval before any external submission scaffold:

| Capability | Details                                                                                                              |
| ---------- | -------------------------------------------------------------------------------------------------------------------- |
| Endpoints  | `GET /compliance/dispute-filing/status`, `GET .../runs`, `POST .../accounts/{id}/prep`, `POST .../runs/{id}/approve` |
| Audit      | `dispute_filing_prep_runs` table (migration `042`)                                                                   |
| Timeline   | Approvals emit `dispute_filing_prep` timeline events on linked cases                                                 |
| Gate       | `ENABLE_DISPUTE_FILING_PREP=true` (requires human-gated agent execution readiness)                                   |
| Deferred   | Autonomous bureau filing, external tool calling without approval, unsupervised agent loops                           |

## Feature flags

| ------------------------------------- | ------------------------------------------ |
| `ENABLE_HRIS_BIDIRECTIONAL_SYNC`      | HRIS sync run audit scaffold               |

| `ENABLE_LLM_DISPUTE_DRAFT_AUGMENT`    | LLM dispute draft augment scaffold         |

| `ENABLE_SAML_FEDERATION_METADATA`     | Required for HRIS sync                     |
| `ENABLE_SMS_MARKETING_DELIVERY`       | Required for deliverability dashboard      |
| `ENABLE_LLM`                          | Required for dispute draft augment         |
| `ENABLE_AGENT_EXECUTION`              | Required for dispute filing prep readiness |
