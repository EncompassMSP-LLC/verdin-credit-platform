# Release Notes — Version 4.3.0 GA (Operational Core)

**Version:** 4.3.0  
**Date:** 2026-06-29  
**Tag:** `v4.3.0`

## Summary

Version 4.3.0 is the **Operational Core** general availability release. It moves Verdin from platform scaffold to a cohesive credit operations product with case management, account intelligence, document processing, event-driven audit, task management, and the **Mission Control** operational dashboard.

## Shipped Capabilities

| Capability                    | Highlights                                                 |
| ----------------------------- | ---------------------------------------------------------- |
| Case Management               | Full CRUD, filters, RBAC, staff UI                         |
| Credit Account Intelligence   | Tradeline scoring, dispute readiness, intelligence summary |
| Document Foundation           | Upload, versioning, MinIO storage, duplicate detection     |
| OCR Pipeline                  | Async worker extraction (pypdf + tesseract)                |
| AI Classification             | Rule-based document type classification                    |
| Metadata Extraction           | Structured field extraction from documents                 |
| Entity Resolution             | Deterministic case/account/person matching                 |
| Timeline & Audit Engine       | Append-only event stream via platform event bus            |
| Task Management               | CRUD, complete/reopen, overdue filters, timeline events    |
| **Mission Control Dashboard** | Single `GET /api/v1/dashboard` product API                 |

## Operational Core Outcome

The platform now supports an end-to-end operational workflow:

```text
Create Case

    ↓

    ↓
Entity Resolution
    ↓

    ↓
Task Management
    ↓

```

## Mission Control Dashboard

The dashboard is a **product**, not a page. One API call returns:

- `overview` — live executive strip
- `cases`, `accounts`, `documents`, `tasks` — domain metrics
- `processing` — document pipeline health
- `performance` — business throughput
- `timeline` — recent activity feed
- `alerts` — OCR failures, unmatched entities, review queue, overdue tasks

The staff UI polls on `refresh_seconds` (30s) and is designed for future WebSocket push without contract changes.
