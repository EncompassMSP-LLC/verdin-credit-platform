# Release Notes — Version 16.0.0 (Reinvestigation Operations & Configuration)

**Version:** 16.0.0  
**Date:** 2026-07-16  
**Tag:** `v16.0.0`

## Summary

Version 16.0.0 advances **Compliance Intelligence Phase 15 — Reinvestigation Operations & Configuration**. It closes the configuration and operations gaps deferred from 5.21 without crossing the legal/compliance-gated frontier: organizations can configure cross-bureau monetary tolerance, operators have an auditable trail for planned bureau response ingestion (no live polling), and reinvestigation outcome analytics gain org-internal trailing baselines for advisory comparison.

Everything remains staff-driven and advisory — the platform never polls a bureau without an explicit future compliance gate, never files a dispute, never escalates without a human, never files suit, and never shares reinvestigation-outcome data across tenants. Live bureau response ingestion execution, automated re-dispute filing execution, unsupervised escalation, automated litigation filing, and cross-tenant benchmarks remain explicitly out of scope. See [`docs/governance/version-16.0-scope.md`](../governance/version-16.0-scope.md).

## Shipped in v16.0.0

| Area                                         | Highlights                                                                                   |
| -------------------------------------------- | -------------------------------------------------------------------------------------------- |
| **Org-configurable cross-bureau tolerance**  | Per-org `cross_bureau_balance_tolerance` (default $1.00) via org-admin dispute settings      |
| **Bureau response ingestion audit scaffold** | Compliance run table + list/status/start API; starts always defer — no live bureau API calls |
| **Org-internal reinvestigation benchmarks**  | Trailing baseline/recent outcome rates + advisory deltas; org-scoped only                    |
| **Governance**                               | Capability matrix 16.0 sign-off, scope doc, completion checklist, release notes              |

## Org-configurable cross-bureau balance tolerance

| Capability | Details                                                                                         |
| ---------- | ----------------------------------------------------------------------------------------------- |
| Endpoints  | `GET` / `PATCH /org-admin/dispute-settings` (admin)                                             |
| Payload    | `cross_bureau_balance_tolerance` (Decimal, default `$1.00`, range `$0.01`–`$100.00`)            |
| Wiring     | Litigation-packet cross-bureau evidence resolves the org value at read time for monetary fields |
| Guardrail  | Admin-only; no live bureau contact; tolerance applies only to monetary cross-bureau fields      |

## Bureau response ingestion audit scaffold

| Capability | Details                                                                                     |
| ---------- | ------------------------------------------------------------------------------------------- |
| Endpoints  | `GET` / `POST` under `/compliance/bureau-response-ingestion` (status, runs list/get, start) |
| Payload    | Run records with scope, status, and deferral reason; start always records `status=deferred` |
| Guardrail  | Staff-mediated; `ready` / live polling stay false; no external bureau API calls             |

## Org-internal reinvestigation outcome benchmarks

| Capability | Details                                                                                                                      |
| ---------- | ---------------------------------------------------------------------------------------------------------------------------- |
| Endpoint   | `GET /reporting/reinvestigation-outcomes/benchmarks` (`baseline_days` default 90, `recent_days` default 30, optional bureau) |
| Payload    | `baseline` / `recent` analytics plus advisory `rate_deltas` (recent − baseline); `scope` is always `organization`            |
| Guardrail  | Org-scoped only (no cross-tenant data); read-only; `recent_days` > `baseline_days` returns `422`                             |

## Testing & CI

- API tests for org-internal benchmarks (baseline vs recent windows, validation)
- `@verdin/api-client` types + helpers for org-admin dispute settings, compliance ingestion, and benchmarks
- UI: Org Admin dispute-settings card for cross-bureau tolerance

## Deferred to 17.0+ (or never)

- Live bureau response ingestion / polling execution (requires live bureau API access + legal/compliance sign-off) — 17.0+
- Automated re-dispute filing execution (human gate retained) — 17.0+
- Unsupervised CFPB / attorney escalation — **never**
- Automated litigation filing / e-filing — **never** (the export is a human handoff; the platform never files)
- Cross-tenant reinvestigation-outcome benchmarks (data governance + legal review) — 17.0+
