# Release Notes — Version 5.4.0 (Production Operations)

**Version:** 5.4.0  
**Date:** 2026-07-04  
**Tag:** `v5.4.0`

## Summary

Version 5.4.0 completes the **Production Operations** milestone after v5.3.0. It ships billing invoicing and dunning scaffolds, multi-IdP federation registry, marketing SMS campaign enqueue, and agent observability with timeline correlation — all behind feature flags and documented **Partial** scope limits.

All four 5.4 epics ship as **Partial ✅**. See [`docs/governance/version-5.4-scope.md`](../governance/version-5.4-scope.md).

## Shipped in v5.4.0

| Area                         | Highlights                                                                                                    |
| ---------------------------- | ------------------------------------------------------------------------------------------------------------- |
| **Billing invoicing**        | Invoice/dunning run audit, status/list/run endpoints behind `ENABLE_BILLING_INVOICING`                        |
| **Identity federation**      | Multi-IdP provider registry, federation status/list/register behind `ENABLE_IDP_FEDERATION`                   |
| **Communications marketing** | Marketing SMS campaign enqueue scaffold + run audit behind `ENABLE_SMS_MARKETING_CAMPAIGNS`                   |
| **AI agent observability**   | Agent run audit log, status/list/run endpoints, case timeline correlation behind `ENABLE_AGENT_OBSERVABILITY` |
| **Governance**               | Capability matrix 5.4 sign-off, scope doc, completion checklist, release notes                                |

## Billing invoicing and dunning

Org-scoped invoice and dunning reminder scaffold when billing is enabled:

| Capability | Details                                                                                       |
| ---------- | --------------------------------------------------------------------------------------------- |
| Endpoints  | `GET /billing/invoicing/status`, `GET /billing/invoicing/runs`, `POST /billing/invoicing/run` |
| Audit      | `billing_invoicing_runs` table (migration `032`)                                              |
| Gate       | `ENABLE_BILLING_INVOICING=true` (requires `ENABLE_BILLING`)                                   |
| Deferred   | Stripe invoice PDF generation, payment collection automation, tax calc                        |

## Multi-IdP federation

Enterprise identity federation provider registry aligned with SCIM provision audit:

| Capability | Details                                                                                                              |
| ---------- | -------------------------------------------------------------------------------------------------------------------- |
| Endpoints  | `GET /enterprise/federation/status`, `GET /enterprise/federation/providers`, `POST /enterprise/federation/providers` |
| Audit      | `idp_federation_providers` table (migration `033`)                                                                   |
| Gate       | `ENABLE_IDP_FEDERATION=true` (requires `ENABLE_ENTERPRISE`)                                                          |
| Deferred   | SAML metadata upload, HRIS bidirectional sync, passwordless enrollment UI                                            |

## Marketing SMS campaigns

Marketing SMS campaign enqueue scaffold with org-scoped run audit:

| Capability | Details                                                                                                                     |
| ---------- | --------------------------------------------------------------------------------------------------------------------------- |
| Endpoints  | `GET /notifications/sms-campaigns/status`, `GET /notifications/sms-campaigns/runs`, `POST /notifications/sms-campaigns/run` |
| Audit      | `sms_marketing_campaign_runs` table (migration `034`)                                                                       |
| Gate       | `ENABLE_SMS_MARKETING_CAMPAIGNS=true` (requires `ENABLE_SMS_DELIVERY`)                                                      |
| Deferred   | Twilio bulk send, deliverability dashboards, multi-provider failover, A/B testing                                           |

## Agent observability

Agent run audit scaffold for staff review without autonomous execution:

| Capability | Details                                                                      |
| ---------- | ---------------------------------------------------------------------------- |
| Endpoints  | `GET /llm/agents/status`, `GET /llm/agents/runs`, `POST /llm/agents/run`     |
| Audit      | `agent_observability_runs` table (migration `035`)                           |
| Timeline   | Case-scoped runs emit `agent_observability_run` timeline events              |
| Gate       | `ENABLE_AGENT_OBSERVABILITY=true` (requires `ENABLE_AI`)                     |
| Deferred   | Autonomous agent execution, external tool calling, dispute filing automation |

## Feature flags

| Flag                             | Purpose                                     |
| -------------------------------- | ------------------------------------------- |
| `ENABLE_BILLING_INVOICING`       | Invoice/dunning run scaffold + staff reads  |
| `ENABLE_IDP_FEDERATION`          | Multi-IdP provider registry scaffold        |
| `ENABLE_SMS_MARKETING_CAMPAIGNS` | Marketing SMS campaign enqueue scaffold     |
| `ENABLE_AGENT_OBSERVABILITY`     | Agent run audit + status/list/run endpoints |
| `ENABLE_BILLING`                 | Required for invoicing scaffold             |
| `ENABLE_ENTERPRISE`              | Required for IdP federation                 |
| `ENABLE_SMS_DELIVERY`            | Required for marketing SMS campaigns        |
| `ENABLE_AI`                      | Required for agent observability            |
