# Release Notes — Version 20.0.0 (Reinvestigation Benchmark Parity)

**Version:** 20.0.0  
**Date:** 2026-07-17  
**Tag:** `v20.0.0`

## Summary

Version 20.0.0 advances **Compliance Intelligence Phase 19 — Reinvestigation Benchmark Parity**. It closes parity gaps on org-internal outcome benchmarks: Reporting Center can break down trailing baselines by recipient (credit bureau vs furnisher) in one call, and staff can download an aggregate rates CSV with **no PII**.

Everything remains staff-driven and advisory — the platform never polls a bureau, never files a dispute, never escalates without a human, never files suit, and never shares reinvestigation-outcome data across tenants. Live bureau response ingestion execution, automated re-dispute filing execution, unsupervised escalation, automated litigation filing, and cross-tenant benchmarks remain explicitly out of scope. See [`docs/governance/version-20.0-scope.md`](../governance/version-20.0-scope.md).

## Outcome benchmarks per-recipient breakdown

| Capability | Details                                                                                    |
| ---------- | ------------------------------------------------------------------------------------------ |
| Surface    | Enterprise reporting → **Outcome benchmarks** → Break down by Recipient                    |
| API        | `GET /reporting/reinvestigation-outcomes/benchmarks?group_by=recipient` → `by_recipient[]` |
| Guardrail  | Org-scoped only; advisory deltas; unlinked responses → `unknown`; invalid `group_by` → 422 |

## Org-internal benchmarks aggregate rates export

| Capability | Details                                                                                          |
| ---------- | ------------------------------------------------------------------------------------------------ |
| Surface    | Enterprise reporting → **Outcome benchmarks** → Download CSV                                     |
| API        | `GET /reporting/reinvestigation-outcomes/benchmarks/export?format=csv`                           |
| Guardrail  | Counts/rates only; same window/`bureau`/`group_by` params as JSON; no client/account identifiers |
