# Release Notes — Version 5.3.0 (Enterprise Depth)

**Version:** 5.3.0  
**Date:** 2026-07-04  
**Tag:** `v5.3.0`

## Summary

Version 5.3.0 completes the **Enterprise Depth** milestone after v5.2.0. It ships billing usage metering, SCIM provisioning, predictive analytics, an internal API developer portal with key rotation, and batch document LLM summarization — all behind feature flags and documented **Partial** scope limits.

All five 5.3 epics ship as **Partial ✅**. See [`docs/governance/version-5.3-scope.md`](../governance/version-5.3-scope.md).

## Shipped in v5.3.0

| Area                     | Highlights                                                                                              |
| ------------------------ | ------------------------------------------------------------------------------------------------------- |
| **Billing metering**     | Usage event recording, `GET /billing/usage/summary`, audit log behind `ENABLE_BILLING_USAGE_METERING`   |
| **Identity (SCIM)**      | SCIM 2.0 Users/Groups provision scaffold, provision audit log behind `ENABLE_SCIM_PROVISIONING`         |
| **Predictive analytics** | Outcome aggregates, snapshot refresh, `GET /reporting/predictive/outcomes` behind dedicated flag        |
| **API integrations**     | Internal developer portal, API key rotation workflow, org admin UI behind `ENABLE_API_DEVELOPER_PORTAL` |
| **LLM operations**       | Batch document summarization enqueue + worker job behind `ENABLE_BATCH_LLM_SUMMARIES` + `ENABLE_LLM`    |
| **Governance**           | Capability matrix 5.3 sign-off, scope doc, completion checklist, release notes                          |

## Billing usage metering

Org-scoped usage event scaffold when billing is enabled:

| Capability | Details                                                          |
| ---------- | ---------------------------------------------------------------- |
| Endpoints  | `GET /billing/usage/summary`, `POST /billing/usage/events`       |
| Audit      | `billing_usage_events` table (migration `027`)                   |
| Gate       | `ENABLE_BILLING_USAGE_METERING=true` (requires `ENABLE_BILLING`) |
| Deferred   | Stripe metered billing sync, invoicing PDFs, dunning automation  |

## SCIM provisioning

Enterprise identity provisioning scaffold aligned with OIDC enrollment:

| Capability | Details                                                             |
| ---------- | ------------------------------------------------------------------- |
| Endpoints  | `GET /enterprise/scim/status`, SCIM 2.0 Users/Groups provision/list |
| Audit      | `scim_provision_logs` table (migration `028`)                       |
| Gate       | `ENABLE_SCIM_PROVISIONING=true` (requires `ENABLE_ENTERPRISE`)      |
| Deferred   | Multi-IdP federation UI, HRIS bidirectional sync, passwordless      |

## Predictive analytics

Org-scoped historical outcome read model for staff reporting:

| Capability | Details                                                                                                        |
| ---------- | -------------------------------------------------------------------------------------------------------------- |
| Endpoints  | `GET /reporting/predictive/status`, `GET /reporting/predictive/outcomes`, `POST /reporting/predictive/refresh` |
| Storage    | `predictive_outcome_snapshots` + refresh runs (migration `029`)                                                |
| Gate       | `ENABLE_PREDICTIVE_ANALYTICS=true`                                                                             |
| Deferred   | Cross-org benchmarks, real-time model serving, autonomous prioritization                                       |

## API developer portal and key rotation

Internal operator surfaces for API key lifecycle:

| Capability | Details                                                                        |
| ---------- | ------------------------------------------------------------------------------ |
| Endpoints  | `GET /org-admin/developer-portal`, `POST /org-admin/api-keys/{id}/rotate`      |
| Audit      | `api_key_rotation_logs` table (migration `030`)                                |
| UI         | `DeveloperPortalCard` on org admin page                                        |
| Gate       | `ENABLE_API_DEVELOPER_PORTAL=true`                                             |
| Deferred   | Public external developer portal, OAuth client credentials, per-route limit UI |

## Batch document LLM summaries

Worker-backed batch summarization for documents with OCR text:

| Capability | Details                                                                  |
| ---------- | ------------------------------------------------------------------------ |
| Endpoints  | `GET /documents/batch-llm-summaries/status`, list runs, `POST .../run`   |
| Worker     | `JobType.BATCH_DOCUMENT_LLM_SUMMARY` with PII scrubbing + timeline audit |
| Audit      | `batch_document_summary_runs` table (migration `031`)                    |
| Gate       | `ENABLE_BATCH_LLM_SUMMARIES=true` + `ENABLE_LLM=true` + ADR-012 policy   |
| Deferred   | LLM dispute draft augment, autonomous agents, external batch export      |

## Feature flags

| Flag                            | Purpose                                     |
| ------------------------------- | ------------------------------------------- |
| `ENABLE_BILLING_USAGE_METERING` | Usage event recording + summary reads       |
| `ENABLE_SCIM_PROVISIONING`      | SCIM 2.0 provision endpoints + audit        |
| `ENABLE_PREDICTIVE_ANALYTICS`   | Predictive outcome aggregates + refresh     |
| `ENABLE_API_DEVELOPER_PORTAL`   | Developer portal + API key rotation         |
| `ENABLE_BATCH_LLM_SUMMARIES`    | Batch document LLM summarization worker job |
| `ENABLE_BILLING`                | Required for usage metering                 |
| `ENABLE_ENTERPRISE`             | Required for SCIM provisioning              |
| `ENABLE_LLM`                    | Required for batch LLM summaries            |
