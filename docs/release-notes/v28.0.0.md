# Release Notes — Version 28.0.0 (Monitoring Report Parser Depth)

**Version:** 28.0.0  
**Date:** 2026-07-22  
**Tag:** `v28.0.0`

## Summary

Version 28.0.0 advances **Compliance Intelligence Phase 27 — Monitoring Report Parser Depth**. It hardens IdentityIQ parsing with a golden PDF / expected-JSON CI regression and adds a **SmartCredit** monitoring / tri-merge parser sibling so CRO staff can import the two most common client-sourced monitoring layouts into per-bureau tradelines.

Everything remains staff-driven and advisory — the platform never polls a bureau, never files a dispute, never escalates without a human, never files suit, and never shares reinvestigation-outcome data across tenants. IdentityIQ/SmartCredit B2B pull APIs remain out of scope. See [`docs/governance/version-28.0-scope.md`](../governance/version-28.0-scope.md).

## IdentityIQ golden fixture regression

| Capability | Details                                                                            |
| ---------- | ---------------------------------------------------------------------------------- |
| Surface    | CI / `apps/api/tests/report_parsers/test_identityiq_regression.py`                 |
| Mechanism  | Golden PDF parsed to `ParsedCreditReport` and compared to checked-in expected JSON |
| Guardrail  | Heuristic OCR aid only; no live IdentityIQ portal pull                             |

## SmartCredit monitoring / tri-merge parser

| Capability | Details                                                                                      |
| ---------- | -------------------------------------------------------------------------------------------- |
| Surface    | Document credit-report parse / reparse (`smartcredit` in registry)                           |
| Mechanism  | Detect SmartCredit branding / portal URL; expand TU/EX/EQ columns into per-bureau tradelines |
| Guardrail  | Report-level bureau `unknown`; empty columns skipped; no live SmartCredit API                |
