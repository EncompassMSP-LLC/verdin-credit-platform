# Release Notes — v4.3 M4: Metadata & Entity Resolution Engine

**Date:** 2026-06-29  
**Branch:** `feature/document-entity-resolution`  
**Epic:** Document Intelligence Platform — Milestone 4

## Summary

Document Intelligence M4 adds rule-based metadata extraction from OCR text and deterministic entity resolution to link documents to Cases and Credit Accounts. Ambiguous matches are flagged for manual confirm/reject in the UI.

## New capabilities

### Shared packages

- **`packages/document-metadata`** — regex/rule extraction for consumer name, bureau, creditor, collection agency, masked account number, dates, balance, payment status, addresses, phone numbers, and masked SSN
- **`packages/entity-resolution`** — resolver registry with case, account, person, and organization matchers

### Backend

- Alembic migration `007_document_metadata_resolution` (`document_metadata`, `document_entity_resolutions`)
- API endpoints:
  - `GET/POST /documents/{id}/metadata`
  - `GET/POST /documents/{id}/resolutions`
  - `POST /documents/{id}/resolutions/{resolution_id}/confirm|reject`
- Worker pipeline: classification → metadata extract → entity resolve
- List filters: `metadata_status`, `resolution_status`

### Frontend

- Document Detail: extracted metadata panel and entity match review
- Document Library: metadata status badge and filters

## Configuration

| Setting                              | Default | Description                                       |
| ------------------------------------ | ------- | ------------------------------------------------- |
| `document_metadata_enabled`          | `true`  | Queue metadata extraction after classification    |
| `document_entity_resolution_enabled` | `true`  | Queue entity resolution after metadata extraction |

## Tests

- Package unit tests: extraction, resolution, ambiguous matches, unmatched fallback
- API integration tests: extract, resolve, confirm, reject, list filters
