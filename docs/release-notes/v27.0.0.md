# Release Notes — Version 27.0.0 (Dispute Playbook Depth & Case Entity Re-resolve)

**Version:** 27.0.0  
**Date:** 2026-07-22  
**Tag:** `v27.0.0`

## Summary

Version 27.0.0 advances **Compliance Intelligence Phase 26 — Dispute Playbook Depth & Case Entity Re-resolve**. It deepens the investigator playbook with finding-panel deep-links and closes the remaining case-scoped entity recovery gap with bulk entity re-resolve enqueue.

Everything remains staff-driven and advisory — the platform never polls a bureau, never files a dispute, never escalates without a human, never files suit, and never shares reinvestigation-outcome data across tenants. Live bureau response ingestion execution, automated re-dispute filing execution, unsupervised escalation, automated litigation filing, and cross-tenant benchmarks remain explicitly out of scope. See [`docs/governance/version-27.0-scope.md`](../governance/version-27.0-scope.md).

## Shipped in v27.0.0

| Area                                  | Highlights                                                                                      |

| **Playbook finding deep-links**       | Navigate from Dispute Playbook issues into Case Detail finding anchors (`source_id` / panels)   |

| **Governance**                        | Capability matrix 27.0 sign-off, scope doc, completion checklist, release notes                 |

## Playbook finding deep-links

| Capability | Details                                                                                                                     |
| ---------- | --------------------------------------------------------------------------------------------------------------------------- |
| Surface    | Case Dispute Playbook → Case Detail finding panels                                                                          |
| Mechanism  | `?finding_source=` query + hash anchors (`#metro2-findings`, `#fcra-findings`, etc.) with row/match highlight where present |
| Guardrail  | Advisory only; no new scoring engine; no auto-file                                                                          |

## Case-level bulk entity re-resolve enqueue

| Capability | Details                                                                                                       |
| ---------- | ------------------------------------------------------------------------------------------------------------- |
| Surface    | Case Documents recovery → Re-resolve (bulk)                                                                   |
| API        | `POST /cases/{id}/resolutions/reresolve`                                                                      |
| Guardrail  | Soft-skips docs without metadata; 503 when entity resolution disabled; staff-mediated; no live bureau contact |
