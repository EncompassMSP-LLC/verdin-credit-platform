# Release Notes — Version 23.0.0 (Document Pipeline Recovery Depth)

**Version:** 23.0.0  
**Date:** 2026-07-21  
**Tag:** `v23.0.0`

## Summary

Version 23.0.0 advances **Compliance Intelligence Phase 22 — Document Pipeline Recovery Depth**. It closes remaining non-blocked document-pipeline recovery gaps: staff can enqueue async metadata re-extract for OCR'd documents (without hanging the API on sync extract), and can bulk re-parse all eligible credit reports on a case from Credit Report History.

Everything remains staff-driven and advisory — the platform never polls a bureau, never files a dispute, never escalates without a human, never files suit, and never shares reinvestigation-outcome data across tenants. Live bureau response ingestion execution, automated re-dispute filing execution, unsupervised escalation, automated litigation filing, and cross-tenant benchmarks remain explicitly out of scope. See [`docs/governance/version-23.0-scope.md`](../governance/version-23.0-scope.md).

## Operator async metadata re-extract

| Capability | Details                                                                             |
| ---------- | ----------------------------------------------------------------------------------- |
| Surface    | Document Detail → Re-extract metadata (when OCR present)                            |
| API        | `POST /documents/{id}/metadata/reextract`                                           |
| Guardrail  | Org-scoped; 422 without OCR; 503 when metadata disabled; sync extract endpoint kept |

## Case-level bulk credit-report re-parse

| Capability | Details                                                                                              |
| ---------- | ---------------------------------------------------------------------------------------------------- |
| Surface    | Case Detail → Credit report history → Re-parse all credit reports                                    |
| API        | `POST /cases/{id}/parsed-credit-reports/reparse`                                                     |
| Guardrail  | Soft-skip missing OCR / wrong type / enqueue failure; queued/skipped counts; no live bureau; no auto |
