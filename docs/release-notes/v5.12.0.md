# Release Notes — Version 5.12.0 (Compliance-Gated Expansion Surfaces)

**Version:** 5.12.0  
**Date:** 2026-07-07  
**Tag:** `v5.12.0`

## Summary

Version 5.12.0 completes the **Compliance-Gated Expansion Surfaces** milestone after v5.11.0. It ships bureau live API invocation audit enrichment, public OAuth developer portal scaffold, cross-org benchmark analytics scaffold, and mobile passkey readiness scaffold — all behind feature flags and documented **Partial** scope limits.

All four 5.12 epics ship as **Partial ✅**. See [`docs/governance/version-5.12-scope.md`](../governance/version-5.12-scope.md).

## Shipped in v5.12.0

| Area                              | Highlights                                                                           |
| --------------------------------- | ------------------------------------------------------------------------------------ |
| **Bureau live API invocation**    | Invocation reference/channel audit fields on operator-gated live API invocation runs |
| **Public OAuth developer portal** | Org-scoped OAuth app registration and admin approval audit scaffold                  |
| **Cross-org benchmark analytics** | Governance-gated aggregate benchmark reads and manual refresh run audit              |
| **Mobile passkey readiness**      | Web-first passkey readiness audit from approved HRIS passwordless UI runs            |
| **Governance**                    | Capability matrix 5.12 sign-off, scope doc, completion checklist, release notes      |

## Bureau live API invocation

Operator-gated bureau live API invocation audit enrichment from approved bureau submission runs:

| Capability | Details                                                                                                                        |
| ---------- | ------------------------------------------------------------------------------------------------------------------------------ |
| Endpoints  | `GET /compliance/bureau-live-api/status`, `GET .../runs`, `POST .../submission-runs/{id}/invoke`, `POST .../runs/{id}/approve` |
| Audit      | `invocation_reference_id` and `invocation_channel` fields on invocation runs (migration `063`)                                 |
| Gate       | `ENABLE_BUREAU_LIVE_API=true` (requires bureau submission readiness)                                                           |
| Deferred   | Fully autonomous bureau API filing, unsupervised live submission loops, automatic retries without operator review              |

## Public OAuth developer portal

Admin-gated OAuth developer portal app registration audit scaffold:

| Capability | Details                                                                                                     |
| ---------- | ----------------------------------------------------------------------------------------------------------- |
| Endpoints  | `GET /org-admin/developer-portal/oauth-apps`, `POST .../oauth-apps`, `POST .../oauth-apps/{app_id}/approve` |
| Audit      | `oauth_developer_apps` table (migration `064`)                                                              |
| Gate       | `ENABLE_PUBLIC_OAUTH_DEVELOPER_PORTAL=true` (requires enterprise identity and API key governance scaffolds) |
| Deferred   | OAuth marketplace publishing, unreviewed third-party access approvals, automated trust scoring              |

## Cross-org benchmark analytics

Governance-gated cross-org benchmark read model and refresh run audit:

| Capability | Details                                                                                                          |
| ---------- | ---------------------------------------------------------------------------------------------------------------- |
| Endpoints  | `GET /reporting/cross-org-benchmarks/status`, `GET .../cross-org-benchmarks`, `GET .../runs`, `POST .../refresh` |
| Audit      | `cross_org_benchmark_runs` table (migration `065`)                                                               |
| Gate       | `ENABLE_CROSS_ORG_BENCHMARK_ANALYTICS=true` (requires governance-approved aggregate benchmark sources)           |
| Deferred   | Raw tenant data export, PII-including benchmark payloads, unrestricted benchmark dimensions                      |

## Mobile passkey readiness

Admin-gated mobile passkey readiness run audit from `approved` HRIS passwordless UI runs:

| Capability | Details                                                                                                                                   |
| ---------- | ----------------------------------------------------------------------------------------------------------------------------------------- |
| Endpoints  | `GET /enterprise/federation/mobile-passkey-readiness/status`, `GET .../runs`, `POST .../ui-runs/{id}/start`, `POST .../runs/{id}/approve` |
| Audit      | `mobile_passkey_readiness_runs` table (migration `066`)                                                                                   |
| Gate       | `ENABLE_MOBILE_PASSKEY_READINESS=true` (requires HRIS passwordless UI readiness)                                                          |
| Deferred   | Native mobile app clients, device attestation rollout, passwordless enforcement without operator gates                                    |

## Feature flags

| Flag                                   | Purpose                                              |
| -------------------------------------- | ---------------------------------------------------- |
| `ENABLE_BUREAU_LIVE_API`               | Bureau live API invocation audit scaffold            |
| `ENABLE_PUBLIC_OAUTH_DEVELOPER_PORTAL` | Public OAuth developer portal app registration audit |
| `ENABLE_CROSS_ORG_BENCHMARK_ANALYTICS` | Cross-org benchmark analytics read model scaffold    |
| `ENABLE_MOBILE_PASSKEY_READINESS`      | Mobile passkey readiness audit scaffold              |
| `ENABLE_DISPUTE_BUREAU_SUBMISSION`     | Required for bureau live API readiness               |
| `ENABLE_API_DEVELOPER_PORTAL`          | Required for public OAuth portal readiness           |
| `ENABLE_HRIS_PASSWORDLESS_UI`          | Required for mobile passkey readiness                |
