# Release Notes — Version 5.2.0 (Deferred Production Surfaces)

**Version:** 5.2.0  
**Date:** 2026-07-04  
**Tag:** `v5.2.0`

## Summary

Version 5.2.0 completes the **Deferred Production Surfaces** milestone started after v5.1.0. It ships production SMS delivery, LLM document summaries with staff UI, real Web Push HTTP for portal messaging, org-scoped revenue analytics from billing state, and Redis-backed API key rate limiting — all behind feature flags and documented **Partial** scope limits.

All five 5.2 epics ship as **Partial ✅**. See [`docs/governance/version-5.2-scope.md`](../governance/version-5.2-scope.md).

## Shipped in v5.2.0

| Area                  | Highlights                                                                                         |
| --------------------- | -------------------------------------------------------------------------------------------------- |
| **Communications**    | Twilio SMS adapter, delivery audit log, notification send endpoints behind `ENABLE_SMS_DELIVERY`   |
| **LLM expansion**     | `POST /documents/{id}/llm-summary`, PII scrubbing, `DocumentLlmSummaryPanel` behind `ENABLE_LLM`   |
| **Portal real-time**  | VAPID Web Push HTTP via `pywebpush` on staff-message dispatch behind `ENABLE_PORTAL_PUSH`          |
| **Revenue analytics** | `GET /reporting/revenue`, billing-derived readiness score, Revenue readiness tab on reporting UI   |
| **API integrations**  | Fixed-window API key rate limits on `GET /reporting/operations` behind `ENABLE_API_KEY_RATE_LIMIT` |
| **Governance**        | Capability matrix 5.2 sign-off, scope doc, completion checklist, release notes                     |

## Production SMS delivery

Twilio SMS alongside existing email delivery when `ENABLE_SMS_DELIVERY=true`:

| Capability | Details                                                                                              |
| ---------- | ---------------------------------------------------------------------------------------------------- |
| Endpoints  | `GET /notifications/sms/status`, `POST /notifications/sms/send`, `GET /notifications/sms/deliveries` |
| Adapter    | Twilio provider with `sms_delivery_logs` audit table                                                 |
| Schema     | Optional `users.phone_number`; migration `026_user_phone_sms_delivery_logs`                          |
| Deferred   | Marketing campaigns, deliverability dashboards, multi-provider failover                              |

## LLM document summaries

Document-level summaries deferred from v5.1.0, alongside case summary UI from 5.0+:

| Capability | Details                                                           |
| ---------- | ----------------------------------------------------------------- |
| Endpoint   | `POST /documents/{document_id}/llm-summary`                       |
| Audit      | Timeline event `DOCUMENT_LLM_SUMMARY_GENERATED`                   |
| UI         | `DocumentLlmSummaryPanel` on staff document detail                |
| Gate       | `ENABLE_LLM=true` + ADR-012 provider policy                       |
| Deferred   | Batch summarization, LLM dispute draft augment, autonomous agents |

## Portal Web Push HTTP

Production HTTP delivery for portal push notifications:

| Capability | Details                                                                  |
| ---------- | ------------------------------------------------------------------------ |
| Provider   | `pywebpush` with VAPID keys when configured                              |
| Dispatch   | Staff replies on linked cases send Web Push; `portal_push_delivery_logs` |
| Gate       | `ENABLE_PORTAL_PUSH` + `ENABLE_CLIENT_PORTAL`                            |
| Deferred   | Native mobile push, advanced subscription management UI                  |

## Revenue analytics

Org-scoped revenue/readiness read model from Stripe billing state:

| Capability | Details                                                             |
| ---------- | ------------------------------------------------------------------- |
| Endpoint   | `GET /reporting/revenue`                                            |
| Metrics    | Subscription state, client/portal counts, heuristic readiness score |
| UI         | Revenue readiness tab on `ReportingCenterPage`                      |
| Gate       | `ENABLE_BILLING=true`                                               |
| Deferred   | Usage metering, invoicing PDFs, cross-org benchmarks                |

## API key rate limiting

Per-organization rate limits on the production API key path:

| Capability | Details                                                             |
| ---------- | ------------------------------------------------------------------- |
| Scope      | `GET /reporting/operations` via API key auth                        |
| Limiter    | Redis fixed-window; `API_KEY_RATE_LIMIT_PER_MINUTE` (default 60)    |
| Status     | `GET /org-admin/api-keys/rate-limit/status` (admin)                 |
| Gate       | `ENABLE_API_KEY_RATE_LIMIT=true`                                    |
| Deferred   | Public developer portal, automated key rotation, per-route limit UI |

## Feature flags

| Flag                        | Purpose                                       |
| --------------------------- | --------------------------------------------- |
| `ENABLE_SMS_DELIVERY`       | Twilio SMS send + audit log                   |
| `ENABLE_LLM`                | Document LLM summaries + case summaries       |
| `ENABLE_PORTAL_PUSH`        | Web Push HTTP delivery for portal messaging   |
| `ENABLE_BILLING`            | Revenue analytics + Stripe billing scaffold   |
| `ENABLE_API_KEY_RATE_LIMIT` | API key rate limiting on reporting operations |
| `ENABLE_CLIENT_PORTAL`      | Required for portal push                      |

Web mirrors: `VITE_ENABLE_CLIENT_PORTAL`, `VITE_ENABLE_PORTAL_PUSH`.
