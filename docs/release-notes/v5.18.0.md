# Release Notes — Version 5.18.0 (Reinvestigation Depth & Litigation Readiness)

**Version:** 5.18.0  
**Date:** 2026-07-15  
**Tag:** `v5.18.0`

## Summary

Version 5.18.0 advances **Compliance Intelligence Phase 11 — Reinvestigation Depth & Litigation Readiness**. It deepens the 5.17 reinvestigation lifecycle: the FCRA §611 clock now runs per sent dispute round, models the 45-day extended window, aggregates outcomes into per-org trend analytics, and assembles an operator-gated litigation-readiness evidence packet for attorney handoff.

Everything remains staff-driven and advisory — the platform never polls a bureau, never files a dispute, and never files suit. Live bureau response ingestion, automated re-dispute filing, unsupervised escalation, automated litigation filing, and cross-tenant benchmarks remain explicitly out of scope. See [`docs/governance/version-5.18-scope.md`](../governance/version-5.18-scope.md).

## Shipped in v5.18.0

| Area                                     | Highlights                                                                                                                   |

| **Litigation-readiness evidence packet** | Operator-gated §611/§623 evidence bundle (letters + responses + clock) with an advisory willful-noncompliance grade          |
| **Governance**                           | Capability matrix 5.18 sign-off, scope doc, completion checklist, release notes                                              |

## Per-letter multi-round reinvestigation clock

| Capability | Details                                                                                                       |
| ---------- | ------------------------------------------------------------------------------------------------------------- |
| Endpoint   | `GET /accounts/reinvestigation-clock?case_id=` (+ readiness/summary reuse the same clock start)               |
| Logic      | Clock start = latest sent `sent_at` (fallback `last_dispute_date`); `dispute_round_count` = sent letter count |
| Guardrail  | Pure read model; only `status == sent` letters with a non-null `sent_at` count; no live bureau contact        |

## Extended 45-day reinvestigation window

| Capability | Details                                                                                                               |
| ---------- | --------------------------------------------------------------------------------------------------------------------- |
| Logic      | `document_extends_window(...)` flags §611(a)(1)(B) when a case/account document is uploaded within the initial window |
| Payload    | `AccountReinvestigationClock.extended`; clock summary `extended_windows`; deadline extends to start + 45 days         |
| Guardrail  | Derived from existing document timestamps (no migration); mailing-day and out-of-window docs do not trigger it        |

## Reinvestigation outcome analytics

| Capability | Details                                                                                                              |
| ---------- | -------------------------------------------------------------------------------------------------------------------- |
| Endpoint   | `GET /reporting/reinvestigation-outcomes`                                                                            |
| Payload    | Per-outcome counts, deletion/verification/correction/favorable/no-response rates, avg/median time-to-response        |
| Guardrail  | Org-scoped only (no cross-tenant benchmarks); read-only aggregate; ignores `no_response` and negative elapsed values |

## Litigation-readiness evidence packet

| Capability | Details                                                                                                                          |
| ---------- | -------------------------------------------------------------------------------------------------------------------------------- |
| Endpoint   | `GET /accounts/{account_id}/litigation-packet` (case_manager+ write; read-only users get `403`)                                  |
| Logic      | Grades willful-noncompliance evidence into `{ eligible, strength (strong/moderate/weak/not_ready), score, indicators, summary }` |
| Payload    | Reinvestigation trail (letters + responses), §611 clock state, latest outcome, advisory recommendation, disclaimer               |
| Guardrail  | Read-only; never drafts pleadings, files suit, e-files, or transmits to a court/bureau/attorney — a licensed attorney decides    |

## Testing & CI

- Unit tests for the extended-window helper, outcome-analytics aggregation, and litigation-readiness grading branches
- API tests for the `sent_at`-keyed clock, 45-day extension, outcome analytics roll-up, and litigation packet (incl. write-permission gate)
- `@verdin/api-client` types + helpers for the extended clock, outcome analytics, and litigation packet
- New UI: `extended`/`extended_windows` badges on the clock panel, a "Reinvestigation outcomes" Reporting Center tab, and an on-demand `AccountLitigationPacketPanel`

## Deferred to 5.19+ (or never)

- Live bureau response ingestion / polling (requires live bureau API access + legal/compliance sign-off) — 5.19+
- Automated re-dispute filing and unsupervised CFPB / attorney escalation (human gate retained) — 5.19+ / never
- Automated litigation filing / e-filing — **never** (the packet is a human handoff; the platform never files)
- Cross-tenant reinvestigation-outcome benchmarks (data governance + legal review) — 5.19+
