# Release Notes — Version 17.0.0 (Reinvestigation Operations Surfaces)

**Version:** 17.0.0  
**Date:** 2026-07-16  
**Tag:** `v17.0.0`

## Summary

Version 17.0.0 advances **Compliance Intelligence Phase 16 — Reinvestigation Operations Surfaces**. It surfaces the Phase 15 operator APIs in staff UI: Enterprise reporting gains an Outcome benchmarks tab for org-internal trailing baselines, and Compliance Center gains a Response ingestion tab for deferred bureau response ingestion audit runs.

Everything remains staff-driven and advisory — the platform never polls a bureau, never files a dispute, never escalates without a human, never files suit, and never shares reinvestigation-outcome data across tenants. Live bureau response ingestion execution, automated re-dispute filing execution, unsupervised escalation, automated litigation filing, and cross-tenant benchmarks remain explicitly out of scope. See [`docs/governance/version-17.0-scope.md`](../governance/version-17.0-scope.md).

## Reporting Center org-internal benchmarks UI

| Capability | Details                                                                                         |
| ---------- | ----------------------------------------------------------------------------------------------- |
| Surface    | Enterprise reporting → **Outcome benchmarks**                                                   |
| API        | `GET /reporting/reinvestigation-outcomes/benchmarks` (`baseline_days`, `recent_days`, `bureau`) |
| Guardrail  | Org-scoped only; read-only; no cross-tenant data; no live bureau contact                        |

## Compliance Center ingestion audit UI

| Capability | Details                                                                                       |
| ---------- | --------------------------------------------------------------------------------------------- |
| Surface    | Compliance center → **Response ingestion**                                                    |
| API        | Status / list / start under `/compliance/bureau-response-ingestion`                           |
| Guardrail  | Start always records `status=deferred`; no external bureau API calls; blockers remain visible |

## Testing & CI

## Deferred to 18.0+ (or never)

- Automated re-dispute filing execution — 18.0+
- Unsupervised CFPB / attorney escalation — **never**
- Automated litigation filing / e-filing — **never**
- Cross-tenant reinvestigation-outcome benchmarks — 18.0+
