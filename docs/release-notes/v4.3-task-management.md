# Release Notes — v4.3 Task Management

**Date:** 2026-06-29  
**Epic:** Operational Core — Task Management

## Summary

Delivers a full operational task work queue for cases, accounts, and documents with RBAC-protected CRUD, lifecycle actions (complete/reopen), rich list filters, timeline audit events, and a React UI.

## New capabilities

### Backend

- Migration `009_task_management` expands `tasks` with `organization_id`, optional entity links (`case_id`, `account_id`, `document_id`), completion metadata, and source tracking
- Status/priority enums aligned to operational vocabulary (`open`, `blocked`, `canceled`, `critical`)
- `POST/GET/PATCH/DELETE /tasks` plus `POST /tasks/{id}/complete` and `POST /tasks/{id}/reopen`
- Timeline events: `TASK_CREATED`, `TASK_UPDATED`, `TASK_COMPLETED`, `TASK_REOPENED`, `TASK_DELETED`

### Frontend

- Tasks list with status/priority/overdue filters
- Task detail with complete, reopen, and delete confirmation
- Create/edit task forms

### Shared packages

- `@verdin/api-client` — full task API surface
- `@verdin/shared` — task status/priority types and labels
- `@verdin/validation` — Zod schemas for task forms
- `verdin-event-types` — expanded task event types

## Next

- Operational Dashboard with live metrics sourced from tasks, cases, documents, and timeline
