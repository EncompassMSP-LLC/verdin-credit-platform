# Release Notes — Version 19.0.0 (Reinvestigation Benchmark Depth)

**Version:** 19.0.0  
**Date:** 2026-07-16  
**Tag:** `v19.0.0`

## Summary

Version 19.0.0 advances **Compliance Intelligence Phase 18 — Reinvestigation Benchmark Depth**. It deepens org-internal outcome benchmarks: organizations can set optional Equifax / Experian / TransUnion window overrides on dispute settings, and Reporting Center Outcome benchmarks compare bureaus in a single call via `group_by=bureau`.

Everything remains staff-driven and advisory — the platform never polls a bureau, never files a dispute, never escalates without a human, never files suit, and never shares reinvestigation-outcome data across tenants. Live bureau response ingestion execution, automated re-dispute filing execution, unsupervised escalation, automated litigation filing, and cross-tenant benchmarks remain explicitly out of scope. See [`docs/governance/version-19.0-scope.md`](../governance/version-19.0-scope.md).

## Per-bureau benchmark window defaults

| Capability | Details                                                                                |
| ---------- | -------------------------------------------------------------------------------------- |
| Surface    | Org Admin → Dispute settings                                                           |
| API        | `GET`/`PATCH /org-admin/dispute-settings` (`reinvestigation_benchmark_bureau_windows`) |
| Guardrail  | Equifax/Experian/TransUnion only; `null` clears; recent ≤ baseline; org-scoped only    |

## Outcome benchmarks per-bureau breakdown

| Capability | Details                                                                              |
| ---------- | ------------------------------------------------------------------------------------ |
| Surface    | Enterprise reporting → **Outcome benchmarks**                                        |
| API        | `GET /reporting/reinvestigation-outcomes/benchmarks?group_by=bureau` → `by_bureau[]` |
| Guardrail  | Org-scoped only; advisory deltas; invalid `group_by` → 422; no live bureau contact   |
